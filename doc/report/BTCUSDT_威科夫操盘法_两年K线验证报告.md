## 2. 验证口径

原文中的威科夫内容分为两类：可由 K 线复验的市场行为，以及对 CM/主力意图的解释。为避免把叙事当证据，本次只验证可重复计算的代理指标：

- 高量：当前成交量 / 20根均量 >= 1.25-1.50。
- 低量：当前成交量 / 20根均量 <= 0.80-0.95。
- 宽幅：当前振幅 / 20根均振幅 >= 1.05-1.25。
- 小实体：实体 / 全振幅 <= 0.25；较强实体：实体 / 全振幅 >= 0.40-0.55。
- Spring：小幅跌破过去20根低点后收回。
- JOC/SOS：收盘突破过去20根高点，且放量、较强实体、收盘靠近高位。
- UT：上冲过去20根高点但收盘跌回。
- SOW：放量、较强实体跌破过去20根低点。
- 结果检验：按周期使用短窗口和中窗口观察后续收盘收益，顺向为正、反向为负。

## 11. 最终取舍

适合进入 BTC 可复用方法论的内容：

- 只把价格、成交量、振幅/速度作为基础信息；指标可用于过滤但不能替代市场行为。
- 供求关系改写为可观察语言：承接、卖压、突破确认、突破失败、缩量回测、放量无进展。
- SC/BC 只作为流程起点，不作为直接入场；二次测试和右手边确认必须保留。
- Spring 必须区分低量候选、放量待确认、失败 Spring；失败后不能补仓摊低成本。
- JOC/SOS 是最可执行的多头核心之一；低量回测主要用于优化风险位置，而不是保证收益。
- UT/SOW、放量跌回区间和高量跌破结构是退出/降仓优先信号。
- 多时间框架必须硬编码：周线背景 > 日线结构 > 4小时执行。
- 进场后 1-5 根必须做危机管理，若市场行为否定进场依据，先退出再复盘。

需要剔除或降级的内容：

- “CM 正在吸筹/派发”不能由普通 K 线直接证明，只能说“出现吸收/派发代理行为”。
- “威科夫不能与任何指标组合”的表述过强；实盘可使用均线、ATR 等作为风险过滤，但交易理由仍必须来自结构和价量。
- “突破时不追，只等回测”应修正为：突破当根可以作为计划触发，但标准仓位优先等低量回测或更小周期确认。
- 选股相对强弱不适用于单一 BTCUSDT；若要验证，需要加入 ETH、SOL、BNB 或股票池横截面。
- 新闻、主力身份、订单流吸收需要额外数据源，不能在本报告中作为硬规则。

## 12. 二次提炼后的完善方向

复查后，最终方法论需要进一步强调以下边界：

- 单根 K 线信号默认降级为观察；只有“背景 + 位置 + 确认 + 风险可控”同时成立，才进入交易计划。
- 样本很少但命中率很高的信号不能按 100% 理解，必须标记为 C 级证据或等待更多样本。
- 简化“日线威科夫组合规则”在本次两年样本中亏损，因此它不能被包装成自动盈利系统，只能作为人工决策框架的验证工具。
- 低量回测的主要价值是优化止损位置和风险收益比，不是保证后续收益。
- 最终交易手册必须加入暂停交易条件、头寸计算公式和实盘前验收标准，否则容易从“方法论”滑回主观看图。

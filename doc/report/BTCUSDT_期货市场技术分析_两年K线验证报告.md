## 2. 验证口径

《期货市场技术分析》是综合框架，不能把所有章节都直接变成同一种回测。本次将可量化内容拆成代理指标：

- 趋势：50/200均线结构、均线斜率、20/55根通道突破。
- 支撑阻力：通道边界、突破后5根内回测是否守住原突破位。
- 回撤：趋势中的23.6%、38.2%、50%、66.6%区域。
- 量价：突破K线成交量相对20根均量的高低。
- 振荡指标：RSI超买超卖、RSI/MACD背离、MACD零轴上下交叉。
- 布林带：20根均线和2倍标准差，低带宽后突破。
- 多时间框架：日线放量突破是否与20周均线方向一致。
- 组合规则：只作为“可执行性检验”，不代表最佳参数。

## 12. 原文关键内容正确性结论

| 原文内容 | 结论 | BTC 验证/修正说明 |
| --- | --- | --- |
| 市场价格以趋势方式演变，顺势优先 | 验证通过 | 50/200均线状态、通道突破和组合规则均可机械化验证；BTC趋势存在，但会被长时间震荡和高回撤打断。 |
| 同一套技术原理可用于不同时间级别 | 部分通过 | 1h、4h、1d、1w均可计算同类信号，但短周期噪声显著更高；执行层和战略层不能等权。 |
| 道氏趋势可用高低点与长期均线确认 | 验证通过 | 50/200均线趋势状态可作为峰谷结构的可复现代理；确认越严格，信号越少但更稳定。 |
| 成交量验证突破和趋势健康 | 部分通过 | 放量突破整体优于缩量突破，但短周期样本仍有噪声；成交量是加权项，不是独立下单理由。 |
| 支撑阻力突破后会发生角色互换 | 部分通过 | 突破后5根内回测守住原位的样本可观察到优势，但数量和周期差异较大，应作为确认流程而非必然规律。 |
| 1/3、1/2、2/3回撤可用于判断趋势强弱 | 部分通过 | 浅/中/深回撤可量化；超过66.6%的回撤应降低原趋势权重，但不能机械判定反转。 |
| 均线系统能提供趋势过滤，但信号滞后 | 验证通过 | 50/200策略和组合策略可降低主观性；滞后与横盘拉锯需要由通道、成交量和止损补足。 |
| 布林带收缩后容易出现方向突破 | 部分通过 | 低带宽后突破具有可验证价值，但方向必须由收盘突破确认，不能提前预测。 |

## 13. 最终取舍

适合进入 BTC 可复用方法论的内容：

- 技术分析作为概率框架，而不是预测工具。
- 周线定背景，日线定计划，4小时定执行。
- 趋势优先：50/200均线、20/55通道、高低点结构共同判断。
- 突破必须看成交量和回测，缩量突破降级。
- 支撑阻力以最近20/55根高低点、前高前低、200日线和整数位为主。
- 回撤用23.6%、38.2%、50%、66.6%做强弱分层，不机械抄底。
- RSI/MACD/布林带只做确认、过滤和预警，不单独下单。
- 多工具确认至少覆盖趋势、位置、量价、动量和风险收益比。
- 每笔交易必须先写入入场、止损、目标、失效条件和账户风险。

需要剔除或降级的内容：

- OI、跨市场、情绪调查、长期周期不能用本次K线数据直接证明。
- 传统跳空理论不适合直接套到连续交易的BTCUSDT。
- 艾略特完整数浪、头肩/三角/旗形的人工形态识别不可直接机械化为硬规则，只保留其结构思想。
- “某信号必然导致反转”的表述全部降级为“提高警惕并等待确认”。

## 2. 验证口径

原文中的 VPA 概念包含两类内容：一类是可由 K 线验证的量价现象，另一类是对“局内人行为”的解释模型。为了避免把叙事当证据，本次只验证可重复计算的代理指标：

- 高量：当前成交量 / 20根均量 >= 1.25-1.50，按场景调整。
- 低量：当前成交量 / 20根均量 <= 0.75-0.85。
- 高实体：实体 / 全振幅 >= 0.60。
- 低实体：实体 / 全振幅 <= 0.25-0.35。
- 长上影/长下影：影线 / 全振幅 >= 0.45。
- 结果检验：按周期使用短窗口和中窗口观察后续收盘收益，顺向为正、反向为负。

## 11. 最终取舍

适合进入 BTC 可复用方法论的内容：

- 成交量只做相对判断：同一交易所、同一交易对、同一周期内比较。
- 每根 K 线先判断实体、影线和收盘位置，再看成交量是确认还是异常。
- 长影线+低实体+高量必须结合趋势位置：上涨后偏卖压警报，下跌后只提示恐慌释放或潜在承接，不能直接抄底。
- 放量突破优先，缩量突破降级；突破后回踩缩量才提升可信度。
- 缩量逆向运动更适合帮助持仓，不适合独立开仓。
- 多时间跨度必须纳入硬规则：周线定背景，日线定计划，4小时定执行。
- 信号必须成组出现，至少有价格结构确认，不能因单根 K 线直接重仓。

需要剔除或降级的内容：

- “局内人正在买/卖”的断言不能由 OHLCV 证明，只能改写为“出现承接/卖压代理信号”。
- VAP、差量、累积差量需要逐价/逐笔/主动买卖数据，不能用普通 Binance K 线直接验证。
- 新闻验证需要新闻时间戳和事件分类，本次不能纳入硬规则。
- “适用于所有市场、所有时间跨度”的表述过强；BTC 短周期噪声高，必须分周期设阈值。

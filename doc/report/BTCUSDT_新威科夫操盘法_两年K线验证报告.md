## 2. 验证口径

《新威科夫操盘法》中的内容被拆成两层验证：第一层是可由 K 线重复计算的市场行为，如放量、缩量、突破、跌破、三高、三低、Spring、JOC、UT/SOW；第二层是对 CM/主力意图、媒体配合、消息链路的解释，这类内容只能作为叙事背景，不能由普通 BTCUSDT K 线直接证明。

核心代理定义：

- 高量：当前成交量 / 20根均量 >= 1.25-1.50。
- 低量：当前成交量 / 20根均量 <= 0.80-0.95。
- 宽幅：当前振幅 / 20根均振幅 >= 1.05-1.25。
- 三高：高点、低点、收盘均高于上一根；三低反向。
- Spring：小幅跌破过去20根低点后收回；TSO 为深度跌破55根低点后收回。
- JOC/SOS：收盘突破过去20根高点，且放量、实体不小、收盘靠近高位。
- 破冰/SOW：放量、宽幅、实体较强地跌破过去20根低点。
- 结果检验：按周期使用短窗口观察后续收盘收益，顺向为正、反向为负。

## 13. 最终取舍

适合进入新文档的内容：

- 供求关系、努力与结果、因果关系、秩序原则保留，但全部改写成可观察行为。
- SC/BC 只作为观察起点，必须等 ST、Spring、JOC 或右手边确认。
- Spring/TSO 必须区分候选、确认、失败；失败后禁止补仓摊低成本。
- SOS/JOC、LPS、缩量回测是 BTC 波段中最容易流程化的多头场景。
- 三高/三低用于描述趋势秩序，但不能脱离周线背景和关键位。
- UT/SOW/破冰/测冰优先作为退出、降仓和禁止追高规则；做空必须另行验证。
- 进场前写计划，进场后 1-5 根执行危机管理，是验证通过的硬规则。

需要剔除或降级的内容：

- “CM 正在吸筹/派发”“主力买通媒体”等不能由普通 K 线证明，只能作为解释语言。
- “最低风险抄底点”不能理解成高胜率保证，只能表示止损位置清晰。
- “突破后一定要等回测”应修正为：突破当根可触发计划，标准仓位优先等回测或小周期确认。
- TSO、测冰、SOT 在两年 BTC 样本中信号稀疏，不能按书面结论机械交易。
- 单一 BTCUSDT 无法验证选股相对强弱；若要验证需加入 ETH、SOL、BNB 等横截面。

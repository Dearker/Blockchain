# 多时间级别支撑阻力位分析通用模板

适用范围: 适用于任何交易对的支撑位和阻力位分析

分析时间级别: 4H、1D、1W（可扩展至其他时间级别）

分析方法: 斐波那契回调/扩展 + 历史高低点识别 + 多级别对比

输出结果验证：支撑位必须小于最新价格，阻力位必须高于最新价格

## 一、分析流程框架

### 1. 数据获取

步骤1: 先从本地文件夹中获取，如果没有数据，则从交易所API获取不同交易对的合约K线数据

- 时间级别: 4H、1D、1W
- K线数量: 4H的300根、1D的180根、1W的100根
- 数据字段: 开盘时间、开盘价、最高价、最低价、收盘价

本地k线数据：读取规则，在D:\kdata\lines目录下，根据不同的交易对创建了不同的文件夹名称，将不同的交易对的k线数据存储在该文件夹下；如果本地数据不是最新数据，则需要从交易所api拉取数据，并更新到本地文件，文件格式为csv

python分析代码：使用本地D:\kdata\code\support_resistance_analysis.py代码直接进行分析即可，不需要再次重新编写分析代码

报告生成地址：在D:\kdata\report目录下，根据不同的交易对创建不同的文件夹名称，在交易对的目录下创建具体的日期时间文件夹，将不同的交易对生成的报告存储在该文件夹下，多次分析的报告文件以日期文件夹进行区分

交易所API示例:

- 币安: https://fapi.binance.com/fapi/v1/klines?symbol={交易对}&interval={时间级别}&limit=500
- 其他交易所: 根据交易所文档调整API接口

### 2. 支撑/阻力位分析框架

#### 2.1 回溯周期设置

| 时间级别 | 回溯周期 | 说明 |
| --- | --- | --- |
| 4H | 300根 | 短期趋势分析 |
| 1D | 180根 | 中期趋势分析 |
| 1W | 100根 | 长期趋势分析 |

#### 2.2 支撑/阻力位识别方法

##### 方法1: 斐波那契回调/扩展位

支撑位:

- 计算区间: 最近显著高点 → 最近显著低点
- 斐波那契位: 23.6%、38.2%、50%、61.8%、78.6%、100%
- 支撑强度: 38.2%-61.8%为强支撑区

阻力位:

- 计算区间: 最近显著低点 → 最近显著高点（向上扩展）
- 斐波那契位: 100%、127.2%、161.8%、200%
- 阻力强度: 127.2%-161.8%为强阻力区

特殊情况: 当当前价格远低于最近高点时:

- 使用斐波那契回调位（从高点向下回调）
- 斐波那契位: 23.6%、38.2%、50%、61.8%、78.6%、100%

##### 方法2: 历史局部低/高点识别

- 识别规则: 前后window根K线中最低/高点
- window参数:
  - 4H级别: window=3
  - 1D级别: window=5
  - 1W级别: window=5
- 合并规则: 价格相近的低/高点（差异小于2%）合并为一个支撑位

##### 方法3: 最近显著低/高点

- 识别方法: 回溯周期内的最低/高价
- 支撑/阻力强度: 最强支撑/阻力（已通过市场验证）

#### 2.3 支撑位强度判断标准

| 支撑强度 | 判断标准 | 标记 |
| --- | --- | --- |
| 极强 ⭐⭐⭐⭐⭐ | 多级别重叠 + 2次以上测试验证 | ⭐⭐⭐⭐⭐ |
| 强 ⭐⭐⭐⭐ | 单级别最近显著低点 + 1次测试 | ⭐⭐⭐⭐ |
| 中 ⭐⭐⭐ | 斐波那契38.2%-61.8%回调位 | ⭐⭐⭐ |
| 弱 ⭐⭐ | 斐波那契23.6%、78.6%回调位 | ⭐⭐ |
| 极弱 ⭐ | 历史局部低点（未测试） | ⭐ |

#### 2.4 阻力位强度判断标准（与支撑位对称）

| 阻力强度 | 判断标准 | 标记 |
| --- | --- | --- |
| 极强 ⭐⭐⭐⭐⭐ | 多级别重叠 + 2次以上测试验证 | ⭐⭐⭐⭐⭐ |
| 强 ⭐⭐⭐⭐ | 单级别最近显著高点 + 1次测试 | ⭐⭐⭐⭐ |
| 中 ⭐⭐⭐ | 斐波那契127.2%-161.8%扩展 | ⭐⭐⭐ |
| 弱 ⭐⭐ | 斐波那契100%、200%扩展 | ⭐⭐ |
| 极弱 ⭐ | 历史局部高点（未测试） | ⭐ |

### 3. 多级别对比分析

#### 3.1 关键支撑/阻力位汇总

对比方法:

1. 列出各时间级别的最强支撑/阻力位
2. 找到价格相近的支撑/阻力位（差异小于2%）
3. 标记为多级别重叠支撑/阻力位

#### 3.2 多级别重叠判断标准

| 重叠级别 | 支撑/阻力强度 | 说明 |
| --- | --- | --- |
| 3级别重叠 ⭐⭐⭐⭐⭐ | 极强 | 4H+1D+1W三个级别重叠 |
| 2级别重叠 ⭐⭐⭐⭐ | 强 | 两个级别重叠 |
| 单级别 ⭐⭐⭐ | 中 | 单级别最近显著高低点 |

### 4. 支撑/阻力位层级划分

#### 4.1 三层防御体系

第一层（短期防守）:

- 价格范围: 当前价格的正负2%-5%
- 强度: 弱支撑/阻力
- 作用: 短期回调/反弹测试位

第二层（中期防守）:

- 价格范围: 当前价格的正负5%-10%（支撑）或正负5%-15%（阻力）
- 强度: 中-强支撑/阻力
- 作用: 中期趋势防守位

第三层（长期防守）:

- 价格范围: 当前价格的正负10%-20%（支撑）或正负15%-100%（阻力）
- 强度: 极强支撑/阻力
- 作用: 长期趋势防守位，跌破/突破后趋势反转

## 三、分析实施步骤

### 步骤1: 数据准备

1. 获取实时K线数据（4H、1D、1W）
2. 确认数据完整性（时间范围、价格范围）
3. 记录当前价格和分析时间

### 步骤2: 单级别分析（每个时间级别独立分析）

1. 计算斐波那契回调/扩展位
2. 识别历史局部高低点
3. 找到最近显著高低点
4. 合并相近的支撑/阻力位
5. 判断支撑/阻力强度
6. 生成单级别支撑/阻力位列表

### 步骤3: 多级别对比

1. 对比各时间级别的最强支撑/阻力位
2. 找到多级别重叠的支撑/阻力位
3. 判断重叠支撑/阻力强度

### 步骤4: 层级划分

1. 划分三层防御体系（短期、中期、长期）
2. 标记关键支撑/阻力位
3. 验证测试历史

### 步骤5: 策略制定

1. 根据支撑/阻力位层级制定交易策略
2. 设定止损止盈位置
3. 制定不同时间级别的操作建议

### 步骤6: 生成报告

1. 按模板结构生成分析报告
2. 核对数据准确性
3. 提供关键判断和推荐策略

## 四、注意事项

### 4.1 使用建议

1. 动态跟踪: 定期更新分析（建议每周更新）
2. 结合其他指标: 结合MACD、RSI、成交量等指标
3. 风险控制: 设置止损，控制仓位
4. 趋势判断: 结合大趋势判断支撑阻力有效性
5. 支撑和阻力位：多个支撑或者阻力位之间的间隔起码要大于1%，间隔太近的多个支撑位没有太大意义

## 五、模板使用流程

1. 复制本模板
2. 替换{交易对}为目标交易对
3. 获取实时K线数据（运行Python脚本）
4. 按模板结构填充数据
5. 生成分析报告
6. 定期更新（建议每周）

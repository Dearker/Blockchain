# BTC 专业投机原理：验证后可复用交易方法论

> 数据基础：Binance `BTCUSDT` 最近两年 K 线，验证区间 2024-06-28 至 2026-06-28。
> 适用对象：BTCUSDT 日线波段交易，周线用于战略过滤，4小时用于执行优化；不构成投资建议。

## 1. 方法论定位

这套方法保留《专业投机原理》中经过 BTC 两年 K 线验证后仍能落地的部分：保障资本、趋势定义、1-2-3 趋势变动、2B 假突破、均线过滤、成交量辅助、走势年龄、风险/回报和交易纪律。

它不预测每一根 K 线，只回答五个问题：

1. 现在属于什么趋势？
2. 趋势是否出现可验证的变化？
3. 入场、止损、目标和失效条件在哪里？
4. 这笔交易是否至少有 1:3 的风险/回报？
5. 如果判断错了，账户最多亏多少？

## 2. 证据边界

本方法只使用 OHLCV K 线。宏观经济、政策、跨市场确认、期权链、盘口订单流、资金费率和交易者心理不能由本次数据直接证明。因此，它们只能进入检查清单，不能冒充回测结论。

## 3. 极简决策树

每天只按下面顺序决策：

1. 先判定周线：周线明显逆向时，日线只允许试探仓或不交易。
2. 再判定日线状态：只在多头趋势、空头趋势、趋势变化确认、震荡边界四类状态中找机会。
3. 再找触发：只接受 2B、完整 1-2-3 后的再触发、窄幅盘整突破。
4. 再算风险：止损写不清、扣除费用滑点后不足 3R、单笔亏损超限，全部放弃。
5. 最后执行：4小时只能优化入场和失败退出，不能反向推翻日线计划。

一句话版本：先过滤方向，再等结构触发，最后用 3R 和账户风险决定能不能下单。

## 4. 三层时间框架

| 层级 | 周期 | 任务 | 核心工具 |
| --- | --- | --- | --- |
| 战略层 | 周线 | 决定做多、做空或观望 | 10周/30周均线、周线高低点、走势年龄 |
| 计划层 | 日线 | 定义趋势、2B、1-2-3、止损和目标 | 200日均线、50日均线、20/55日高低点、成交量 |
| 执行层 | 4小时 | 优化触发和失败退出 | 20/55根高低点、2B触发、三根/四根反转 |

日线是主战场，周线只决定仓位上限，4小时只优化执行。不要用 1小时噪声推翻日线计划。

## 5. 市场状态

每天先分类，再找机会：

| 状态 | 定义 | 行动 |
| --- | --- | --- |
| 多头趋势 | 日收盘在200日线上方，50日线走平或上行，20/55日高低点抬高 | 只找做多2B、回踩转强或突破计划 |
| 空头趋势 | 日收盘在200日线下方，50日线走平或下行，20/55日高低点下移 | 现货空仓；合约只允许小风险顺势做空 |
| 趋势变化中 | 均线被破坏，且出现失败创新高/低或反向通道突破 | 等1-2-3确认，降低原方向仓位 |
| 震荡整理 | 价格反复穿越50/200日线，20/55日通道无延续 | 只做边界2B，或等待突破 |
| 高波动失控 | ATR抬升、长影线密集、放量失败频繁 | 暂停新仓或降至试探仓 |

验证结果提示：`1d 200日均线多头过滤` 总收益 -16.46%，最大回撤 -31.05%；`1w 10周/30周双向过滤` 总收益 +19.78%。均线能过滤环境，但不能单独入场。

### 5.1 硬定义

| 名词 | 可执行定义 |
| --- | --- |
| 周线明显逆向 | 做多时：周收盘低于10周和30周均线，且10周均线低于30周均线；做空反向 |
| 50日线走平或上行 | 当前50日线 >= 10日前50日线；下行反向 |
| 高低点抬高 | 最近20日高点和20日低点均高于上一组20日高低点 |
| 高波动失控 | 日线振幅超过20日ATR的2.5倍，或连续3日出现长影线且成交量高于20日均量1.3倍 |
| 放量 | 当日成交量 >= 20日均量的1.15倍；缩量为 <= 0.85倍 |
| 标准仓 | 按账户权益0.5%-1.0%计划亏损计算出的头寸，不是满仓 |
| 试探仓 | 标准仓的25%-50%，用于信号合格但背景不完美的场景 |

## 6. 1-2-3 趋势变动

把原书手工趋势线改成可复现版本：

| 条件 | BTC执行定义 |
| --- | --- |
| 1. 趋势线被突破 | 收盘跌破/收回50日线，且50日线斜率不再支持原趋势 |
| 2. 失败创新极值 | 多头不再突破前20/55日高点，或空头不再跌破前20/55日低点 |
| 3. 反向结构突破 | 多头中收盘跌破前20日低点，或空头中收盘突破前20日高点 |

执行规则：

1. 只出现条件1：预警，减仓或收紧止损，不反手。
2. 出现条件1+2：趋势变化概率提高，停止原方向加仓。
3. 三个条件全部出现：确认原趋势可能结束，但不直接开仓；按新方向等待2B、突破或回踩确认。
4. 若确认信号与周线方向冲突，只允许半仓以内，且目标先看2R。

## 7. 2B 假突破

2B 是本方法的核心触发，但必须从“图形感觉”改成订单：

| 方向 | 条件 | 入场 | 止损 |
| --- | --- | --- | --- |
| 顶部2B做空 | 价格突破前20/55日高点，3-5根内不能延续，并收盘跌回前高下方 | 跌回前高下方的收盘或下一根回抽失败 | 假突破最高点上方 |
| 底部2B做多 | 价格跌破前20/55日低点，3-5根内不能延续，并收盘收回前低上方 | 收回前低上方的收盘或下一根回踩守住 | 假突破最低点下方 |

硬门槛：

1. 入场到止损的距离必须清楚。
2. 第一目标至少 3R；若只能到 2R，仓位减半。
3. 周线明显逆向时，不做日线2B重仓。
4. 假突破极值过远，导致止损超过账户风险上限，直接放弃。
5. 触发后若再次突破假突破极值，立即认错。

本次最优代理为 `1d 20日2B/5日确认/持有20日/3R`：交易数 44，胜率 43.18%，平均R 0.33，利润因子 1.60。这说明 2B 不是“每次都对”，而是“错了小亏，对了按 R 倍数兑现”。

### 7.1 2B 取舍表

| 等级 | 条件 | 动作 |
| --- | --- | --- |
| A级 | 周线不逆向，日线位于关键20/55日高低点，假突破后5日内收回，止损距离小于2ATR，目标至少3R | 标准仓 |
| B级 | 周线中性或震荡边界，2B成立但目标只有2R-3R，或成交量没有明显确认 | 试探仓 |
| C级 | 周线逆向、止损超过2ATR、目标不足2R、假突破后横盘拖延超过5日 | 放弃 |

## 8. 窄幅盘整与突破

BTC 版本的窄幅盘整：

1. 日线 15-20 根 K 线内，高低点振幅不超过约 5%。
2. 成交量收缩，波动率下降。
3. 收盘突破区间上沿/下沿。
4. 放量突破优先；缩量突破只观察。
5. 突破后 1-5 根内跌回区间，按失败突破处理。

窄幅盘整突破不是追高理由，而是承接/出货的证据。有效突破必须和周线方向、成交量、止损距离同时合格。

## 9. 成交量

成交量只做辅助确认：

| 场景 | 正面信号 | 负面信号 |
| --- | --- | --- |
| 上涨趋势 | 上涨放量、回调缩量 | 上涨缩量、下跌放量 |
| 下跌趋势 | 下跌放量、反弹缩量 | 下跌缩量、反弹放量 |
| 顶部2B | 突破前高量能不足，跌回前高时放量 | 放量强突破后仅小幅回落 |
| 底部2B | 跌破前低量能衰竭，收回前低时放量 | 放量破位后反抽无量 |

由于只使用 Binance 单一现货成交量，成交量不能单独开仓，只能决定“标准仓、半仓、放弃”。

## 10. 走势年龄

市场走势像有寿命的事件。执行时记录当前趋势从确认日起的两个指标：

| 指标 | 用法 |
| --- | --- |
| 持续时间 | 越接近近两年同类走势的75/90分位，越不追新仓 |
| 累计幅度 | 达到高分位后，新增仓位降级，已有仓位移动止损 |

只有同周期历史走势段不少于 20 段时，才使用 75/90 分位作为定量阈值；样本不足时只做定性提醒。

当日线趋势已经充分延伸，不再用“趋势很好”作为加仓理由。趋势越成熟，新增风险越贵。

## 11. 仓位与风险

先定义亏损，再计算数量：

| 项目 | 规则 |
| --- | --- |
| 单笔计划亏损 | 账户权益的0.5%-1.0%，高波动期降至0.25%-0.5% |
| 最大风险/回报 | 至少1:3；更好是1:5 |
| 同向总风险 | 不超过账户权益的2%-3% |
| 止损位置 | 2B极值、1-2-3失效点、区间内侧或2ATR外 |
| 加仓 | 只加盈利仓；亏损仓绝不摊平 |
| 杠杆 | 强平价必须远离结构止损，否则不用杠杆 |

头寸公式：

| 步骤 | 公式 |
| --- | --- |
| 账户可亏金额 | 账户权益 × 单笔风险比例 |
| 每BTC风险 | 入场价 - 止损价，做空反向计算 |
| 可交易数量 | 账户可亏金额 / 每BTC风险 |
| 加仓后总风险 | 所有仓位到各自止损的亏损合计 |

费用和滑点缓冲：

1. 现货交易按双边手续费和滑点预留至少0.10R成本。
2. 合约交易还要预留资金费率和强平距离，强平价必须远离止损价。
3. 计算后实际目标不足3R时，降级为试探仓或放弃。
4. 止损不能因为费用增加而后移，只能减少数量。

## 12. 入场评分

硬门槛先于评分：

1. 止损价必须能写清。
2. 第一目标至少 3R。
3. 周线不能明显逆向。
4. 价格不能处于高波动失控状态。
5. 亏损仓不能加码。

通过硬门槛后再评分：

| 项目 | 分值 | 条件 |
| --- | --- | --- |
| 周线背景 | 2 | 周线同向或至少不逆向 |
| 日线状态 | 2 | 处于明确趋势、趋势变化确认或清晰区间边界 |
| 触发质量 | 2 | 2B、完整1-2-3后的再触发、放量区间突破之一 |
| 风险/回报 | 2 | 目标≥3R，止损距离合理 |
| 成交量 | 1 | 与信号方向一致 |
| 4小时执行 | 1 | 4小时未出现失败突破，且结构同向 |

执行阈值：

- 8-10分：标准仓位。
- 6-7分：半仓或试探仓。
- 低于6分：不交易。
- 任何硬门槛失效，评分自动作废。

评分降级规则：

1. 周线逆向，最高只能按试探仓。
2. 日线走势年龄样本不足时，不加分，只做风险提示。
3. 成交量与方向相反，至少降一级。
4. 入场后3根日线内没有朝预期方向推进，止损不得放宽。

## 13. 离场

硬退出：

1. 价格触及预设止损。
2. 2B 后重新突破假突破极值。
3. 1-2-3 反转条件被反向破坏。
4. 放量突破后 3-5 根内跌回区间。
5. 心存怀疑且无法用规则解释持仓理由。

盈利保护：

1. 到达 2R 后，止损至少移到入场价或结构保护位。
2. 到达 3R，了结 1/3 至 1/2，剩余仓位跟随趋势。
3. 趋势继续延伸时，只顺着盈利方向移动止损。
4. 出现四天反转、三天高低价反向突破、放量长影线时，先减仓或收紧止损。

## 14. 日常流程

1. 周线：判断10/30周均线、周线高低点、走势年龄。
2. 日线：标记200日线、50日线、20/55日高低点和窄幅区间。
3. 状态：归类为多头、空头、趋势变化、震荡或高波动失控。
4. 信号：只寻找2B、完整1-2-3后的再触发、窄幅盘整突破。
5. 风险：写出入场、止损、3R目标、账户亏损金额。
6. 执行：用4小时优化触发，不用它推翻日线方向。
7. 管理：触发止损立即离场；盈利后移动止损。
8. 复盘：记录是否遵守计划、是否摊平亏损、是否把盈利变亏损。

## 15. 交易工单模板

每笔交易必须先写工单，再下单：

| 字段 | 填写内容 |
| --- | --- |
| 市场状态 | 多头/空头/趋势变化/震荡/高波动失控 |
| 信号类型 | 2B、1-2-3后再触发、窄幅盘整突破 |
| 周线背景 | 同向/中性/逆向 |
| 入场触发 | 具体价格和触发K线 |
| 初始止损 | 价格、依据、距离百分比、距离ATR倍数 |
| 第一目标 | 价格、R倍数、是否扣除费用滑点后仍>=3R |
| 仓位 | 标准仓/试探仓，账户计划亏损金额 |
| 失效条件 | 触发后几根K线内不延续、跌回区间、再破假突破极值等 |
| 管理计划 | 到2R如何移动止损，到3R如何减仓 |

## 16. 复盘字段

复盘只看能改进的行为：

| 字段 | 目的 |
| --- | --- |
| 是否按工单入场 | 检查计划一致性 |
| 是否提前/滞后入场 | 识别执行偏差 |
| 是否按初始止损退出 | 检查是否违反鳄鱼原则 |
| 最大有利/不利波动 | 判断止损和目标是否合理 |
| 实际R倍数 | 统一比较不同价格阶段的交易 |
| 违反规则项 | 只记录事实，不找借口 |
| 下次改进 | 只写一条最重要的改进动作 |

## 17. 禁用规则

- 不因价格“便宜”而买入，也不因价格“太高”而做空。
- 不在没有止损价的情况下开仓。
- 不为了加大仓位而缩窄止损。
- 不在亏损头寸上加码。
- 不把成交量、均线、四天准则、三天准则当作独立重仓信号。
- 不在周线逆向时做日线重仓。
- 不在趋势已经进入高分位延伸后追新仓。
- 不用“这次例外”覆盖交易计划。

## 18. 交易前检查清单

- [ ] 当前趋势和交易周期已定义。
- [ ] 信号属于2B、完整1-2-3后的再触发或窄幅盘整突破。
- [ ] 周线没有明显逆向。
- [ ] 入场价、止损价、3R目标已写清。
- [ ] 单笔亏损不超过账户权益0.5%-1.0%。
- [ ] 成交量没有明显反证。
- [ ] 4小时执行结构没有失败突破。
- [ ] 已填写交易工单。
- [ ] 扣除费用和滑点后仍满足风险/回报要求。
- [ ] 若到达2R，止损移动方案已写好。
- [ ] 若触发止损，会立即离场，不解释、不摊平。

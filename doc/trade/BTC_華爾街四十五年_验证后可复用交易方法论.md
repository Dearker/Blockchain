# BTC 江恩《華爾街四十五年》验证后可复用交易方法论

> 标的：`BTCUSDT`  
> 数据基础：Binance 最近两年 K 线，验证区间 2024-06-27 至 2026-06-27。  
> 适用对象：BTC 现货或低杠杆趋势/波段交易。本文不构成投资建议。

## 1. 方法论定位

江恩方法在 BTC 上可复用的核心不是“精确预测”，而是把交易变成一套有失效点的流程：先保护资本，再顺势交易，用时间和价格寻找候选窗口，用止损和仓位控制人性弱点。

本方法只保留两年 BTC K 线验证后仍有执行价值的内容：资金纪律、止损、趋势方向、百分比位、时间窗、3日图/摆动图、双顶双底、成交量确认和复盘纪律。股票市场专属规则、固定点数、长期季节性和期权权证规则不进入 BTC 执行体系。

验证截止日快照：收盘价 60,429.78，50日均线 69,898.95，200日均线 75,882.88，14日ATR 2,264.25，背景判定：震荡或偏弱。该快照只用于说明验证结束时的市场状态，不是本文方法论的长期结论。

## 2. 一页执行版

每天只按以下顺序做判断：

1. 判断大方向：周线和日线高低点是否同向，价格是否在 200 日线上方。
2. 找候选区：最近主摆动的 25%、33.3%、50%、66.7% 区间，或 18-21天、42-49天、57-65天时间窗。
3. 等市场确认：3日图转强/转弱、突破后回踩不破、双底/双顶完成、成交量确认。
4. 计算交易：止损放在结构位或 1.5-3倍ATR 外，第一目标至少 1.5R。
5. 决定仓位：信号评分 8 分以上标准仓，6-7 分试探仓，低于 6 分不交易。
6. 执行退出：跌破结构止损、3日图反向、关键位有效失守，立即退出；盈利后只顺盈利方向移动止损。

硬过滤条件：

- 没有明确止损，不交易。
- 风险收益比低于 1.5R，不交易。
- 日线空头且没有完成第二低点确认，不做趋势多单。
- 信号只来自时间窗、季节性或百分比位，不交易。
- 连续 3 笔亏损后暂停新开仓，只复盘。

## 3. 第一原则：资本安全优先

江恩最可复用的规则是资金纪律。BTC 波动大，执行时用“账户风险”替代书中的股票点数：

1. 单笔最大亏损控制在账户权益的 0.5%-2%。
2. 单次计划仓位不超过账户权益的 10%-30%；只有周线、日线同向且已有浮盈保护时才允许提高。
3. 入场前必须写明止损价；没有止损价就没有交易。
4. 止损只允许顺盈利方向移动，不能因为希望反弹而取消。
5. 连续 3 笔亏损后暂停新开仓，只做复盘。

本次样本中，买入持有总收益 0.00%、最大回撤 -52.03%；“突破+3ATR移动止损”总收益 21.19%、最大回撤 -22.65%。收益高低不是唯一目标，回撤控制才是江恩方法进入 BTC 的第一过滤器。

## 4. 多周期分工

| 周期 | 职责 | 江恩工具的 BTC 改写 |
| --- | --- | --- |
| 周线 | 判断大方向 | 高低点是否上移、是否站上长期均线、重要百分比位 |
| 日线 | 制定交易计划 | 3日图、7%-9%摆动图、50%回撤、时间窗、成交量 |
| 4小时 | 执行与止损 | 突破/回踩、前高前低、ATR止损、假突破确认 |
| 1小时 | 微调 | 只用于优化入场，不改变日线计划 |

原则：大周期决定是否允许做多，小周期只负责更好的价格和更小的止损。

## 5. 趋势判定

BTC 趋势用三层确认：

1. 高低点：最近两组日线摆动高点和低点同时上移，判定为多头；同时下移，判定为空头或空仓。
2. 均线背景：价格在 200 日线上方且 50 日线不下行，优先寻找多单；价格在 200 日线下方，默认观望。
3. 3日图：连续 3 根日线高低点上移，视为小趋势转强；连续 3 根高低点下移，视为小趋势转弱。

执行规则：

- 多头趋势中只做突破、回踩和二次确认，不猜顶。
- 空头趋势中不因为价格“便宜”补仓。
- 趋势不明时降低交易频率，不用 1 小时级别强行找机会。

## 6. 固定点数的 BTC 改写

江恩书中的“3-5点止损”“9点转向”“18-21点运动”来自股票指数时代，不能直接套到 BTC。

| 原规则 | BTC 改写 |
| --- | --- |
| 3-5点止损 | 1.5-3倍ATR，或结构低点/高点外侧 |
| 9点转向图 | 7%-9% ZigZag 摆动，或 2-3倍ATR摆动 |
| 18-21点运动 | 14%-21%波段，结合前高前低和ATR |
| 突破5点确认 | 突破关键位后收盘超过 0.5-1倍ATR，或超过 1%-2%并回踩不破 |

不能完成这种换算的规则，不进入实盘。

## 7. 百分比位用法

BTC 保留 25%、33.3%、50%、66.7% 四类百分比位，其中 50%最重要。本次 50%近似事件样本 3 个，成功率 100.00%。由于样本很少，这个结果只能说明 50% 附近值得观察，不能理解成稳定胜率。

使用步骤：

1. 找出最近一段日线主升/主跌摆动的起点和终点。
2. 计算 25%、33.3%、50%、66.7% 回撤/反弹位。
3. 价格到位后不直接交易，等待 4小时或日线出现收回、假跌破/假突破、放量突破、3日图转强/转弱。
4. 止损放在结构低点/高点之外，或用 ATR 换算。

百分比位是候选区域，不是命令。若价格直接跌破 50%且无法收回，说明趋势弱，放弃抄底。

## 8. 时间窗用法

本次日线摆动腿落入江恩时间窗的比例为 48.00%，摆动腿中位数为 14 天。时间窗可保留为观察工具：

| 时间窗 | BTC 用法 |
| --- | --- |
| 7-12天 | 短线过热或反弹衰竭观察 |
| 18-21天 / 3周 | 趋势内第一段调整/反弹窗口 |
| 28-31天 | 月度节奏观察，不单独交易 |
| 42-49天 | 6-7周中期调整/反弹窗口 |
| 57-65天 | 重要中期窗口，等待结构确认 |
| 85-92天以上 | 大级别时间窗口，只做预警 |

执行规则：时间到了只代表“开始注意”，不代表立刻买卖。必须等价格失衡或结构信号。

三类调整样本：

- 14-25天：样本 4，正向占比 100.00%
- 42-65天：样本 2，正向占比 100.00%
- 85-92天：样本 0，正向占比 0.00%

样本警戒：以上样本数偏少，只能说明这些时间窗适合“提高注意力”，不能说明时间到了就会转势。时间窗永远只是候选条件，必须由价格结构确认。

## 9. 最安全买点与卖点

最安全买点：

1. 日线已出现一段下跌或调整。
2. 价格形成低点后反弹，再次回调不破前低，形成更高低点。
3. 价格突破第一次反弹高点。
4. 4小时回踩不破突破位，或日线收盘站稳。
5. 止损放在第二个低点下方。

最安全卖点：

1. 长期上涨后出现快速下跌。
2. 反弹形成较低高点。
3. 再次下跌并跌破第一次下跌低点。
4. 若持有多单，清仓；若允许做空，止损放在第二个高点上方。

这套规则的本质是“不要猜第一高低点，等第二次确认”。

## 10. 双顶双底和反复测试

双底/三重底只在趋势背景允许时使用：

- 同一价格区间至少两次获得支撑，容差可用 2%-4% 或 1倍ATR。
- 第二/第三次测试后出现收回和放量确认。
- 突破颈线或第一次反弹高点后入场。
- 止损放在最低支撑下方。

第四次触及同一水平时，不再默认支撑/阻力有效。BTC 反复测试会消耗挂单，第四次更容易出现突破或假突破，必须等待收盘确认。

## 11. 成交量用法

成交量只作为确认项：

- 放量突破 20 日高点后，未来 20 日延续上涨比例为 66.67%。
- 上涨后巨量不等于必然见顶，但提示追高风险。
- 下跌后缩量不等于底部，必须等价格重新站上关键位。
- Binance 成交量不等于全市场成交量，重要时结合其他交易所、ETF资金、期货持仓和清算数据。

执行口径：

1. 突破关键位时，成交量高于过去 90 日 70%-95% 分位，信号加分。
2. 放量突破后无法站稳，视为假突破，优先减仓。
3. 缩量阴跌不抄底，等 3日图或摆动图转强。

## 12. 入场评分

| 项目 | 分值 | 条件 |
| --- | --- | --- |
| 大趋势 | 2 | 周线/日线高低点上移，价格在200日线上方 |
| 3日图 | 1 | 连续3根日线高低点上移，或回调后重新转强 |
| 百分比位 | 1 | 价格在25%、33.3%、50%、66.7%关键位附近获得确认 |
| 时间窗 | 1 | 调整/反弹进入18-21天、42-49天或57-65天窗口 |
| 结构 | 2 | 更高低点后突破前高，或双底/三重底突破 |
| 成交量 | 1 | 突破或收回关键位时有成交量确认 |
| 风险收益比 | 2 | 第一目标至少 1.5R，最好 2R以上 |

执行阈值：

- 8-10分：标准仓位。
- 6-7分：半仓或试探仓。
- 低于6分：不交易。
- 任一硬条件不满足：无止损、止损过远、日线空头明显、风险收益比不足，直接放弃。

## 13. 仓位与止损

仓位计算：

1. 账户可亏金额 = 账户权益 × 单笔风险比例。
2. 每 BTC 风险 = 入场价 - 止损价。
3. 可买数量 = 账户可亏金额 ÷ 每 BTC 风险。
4. 若使用杠杆，强平价必须远离结构止损；否则降低杠杆或放弃。

止损位置优先级：

1. 结构低点/高点外侧。
2. 1.5-3倍ATR。
3. 关键百分比位被有效跌破/突破后的反向收盘。
4. 3日图反向信号。

盈利保护：

- 到 1R 后，把止损至少移动到结构保护位。
- 到 1.5R-2R 后，减仓或移动止损至接近成本。
- 趋势仓用 3ATR 或前一日线摆动低点跟踪。

## 14. 状态机

把交易状态固定为四种，避免临盘随意解释信号：

| 状态 | 条件 | 动作 |
| --- | --- | --- |
| 空仓观察 | 日线空头、趋势不明、评分低于6分 | 只画关键位和时间窗，不下单 |
| 试探仓 | 阶段转强但周线未完全确认，评分6-7分 | 仓位 20%-40%，止损更紧，盈利后快速保护 |
| 标准趋势仓 | 周线/日线同向，评分8分以上，至少1.5R | 按计划仓位执行，用结构位或3ATR跟踪 |
| 防守退出 | 3日图反向、关键位失守、成交量假突破、连续亏损 | 降仓或清仓，禁止加码摊平 |

状态切换规则：

- 观察到试探：必须出现第二低点不破、收回关键百分比位、或3日图转强。
- 试探到趋势：必须突破第一次反弹高点并回踩不破。
- 趋势到防守：价格跌破前一日线摆动低点，或3日图连续转弱。
- 防守到观察：清仓后至少等待一根日线确认，不立刻反手。

## 15. 禁用规则

- 不使用固定美元点数作为 BTC 止损或转向标准。
- 不依据季节性日期直接开仓。
- 不因为“第四次一定突破”提前满仓押方向。
- 不因价格低而买，不因价格高而卖。
- 不在亏损后加码摊平。
- 不撤销止损，不用追加保证金拯救错误交易。
- 不把单一 Binance 成交量当作全市场真相。

## 17. 复盘与迭代

每 20 笔交易或每月复盘一次，只回答五个问题：

1. 亏损是否来自低于 6 分的交易？
2. 是否在日线空头里做了趋势多单？
3. 是否把百分比位或时间窗当成了单独开仓理由？
4. 是否移动或取消了原始止损？
5. 盈利交易是否按 1R、1.5R-2R 规则保护了利润？

迭代原则：

- 只优化造成重复亏损的规则，不因为一笔亏损改系统。
- 若同一类错误连续出现 3 次，把它写入禁用规则。
- 若某类信号连续 20 笔样本后仍不能改善回撤或收益，降级为观察项。
- 新增规则必须能写成“条件、触发、止损、退出、复盘字段”，否则不进入方法论。

## 19. 一句话版本

BTC 上的江恩方法应被改写为：用周线和日线确定趋势，用百分比位和时间窗寻找候选区域，用3日图、摆动高低点和成交量等待确认，用ATR/结构止损和账户风险控制执行；时间和价格只给窗口，真正下单必须由市场确认。

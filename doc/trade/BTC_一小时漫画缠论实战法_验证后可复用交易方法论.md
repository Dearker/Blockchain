# BTC 《一小时漫画缠论实战法》验证后可复用交易方法论

> 标的：BTCUSDT  
> 数据验证区间：2024-06-27 至 2026-06-27  
> 数据来源：Binance BTCUSDT Kline  
> 适用对象：BTC 现货、低杠杆趋势/波段交易；高杠杆短线只可用作辅助框架。  
> 核心原则：先分解结构，再等待确认；只交易完成或接近完成的走势，不交易想象中的走势。

## 1. 保留、修正、删除

保留：

- K 线包含处理、顶底分型、笔、中枢、走势类型。
- 背驰作为走势衰弱预警。
- 区间套作为大级别信号的小级别确认工具。
- 三类买卖点作为入场、加仓、减仓和失效管理框架。
- 买卖同级别，避免用小级别波动破坏大级别持仓计划。

修正：

- 股票基本面筛选改为 BTC 环境筛选：宏观流动性、ETF/机构资金、美元与利率、风险资产联动、链上筹码、合约杠杆。
- “一买 1 成、二买 3 成、三买 6 成”改为账户风险和信号等级决定仓位。
- “三买基本一定参与”改为只在 A/B 级信号中参与。
- “不看 60 分钟”在 BTC 中落地为不混用相邻噪音周期；默认用 `1w/1d/4h/30m/5m`。

删除：

- 固定胜率、固定概率、必然反转、必然趋势延续。
- 没有失效位的“走势必完成”式交易承诺。
- 任何脱离止损距离的固定仓位表。

## 2. 周期配置

| 角色 | 周期 | 用途 |
| --- | --- | --- |
| 环境级别 | 1w | 判断大趋势和是否处在大级别中枢上/下方 |
| 操作级别 | 1d | 决定主要交易方向、核心买卖点和仓位上限 |
| 确认级别 | 4h | 确认日线买卖点，判断背驰是否被小级别承接 |
| 执行级别 | 30m | 找入场触发、止损点和加减仓节奏 |
| 精细级别 | 5m | 只用于优化挂单和短线止损，不反向推翻日线判断 |

本次结构样本：

| 周期 | 中枢数 | 笔数 |
| --- | --- | --- |
| 5m | 1528 | 7695 |
| 30m | 366 | 1790 |
| 4h | 60 | 355 |
| 1d | 12 | 81 |
| 1w | 4 | 13 |

默认用日线做操作级别。只有能稳定盯盘并接受更高噪音时，才把 4h 降为操作级别；30m/5m 只做执行。

## 3. 每次分析的固定流程

1. 先定级别：本次交易到底按日线、4h，还是 30m 执行。
2. 处理包含关系：只保留高低点区间，方向向上取高高/高低，方向向下取低高/低低。
3. 找顶底分型：分型只用于画笔，不直接交易。
4. 画笔：相邻顶底连接，过滤过短噪音笔。
5. 画中枢：连续三笔或三段重叠，记录 `ZD/ZG/DD/GG`。
6. 定当前位置：中枢内、离开中枢、回试中枢、跌回/涨回中枢失效。
7. 看背驰：只比较中枢进入段、离开段和后续同向段，不拿中枢内部乱震比较。
8. 用区间套确认：日线买卖点必须看 4h，4h 买卖点必须看 30m。
9. 先写失效位和仓位，再决定是否下单。

### 3.1 可执行决策树

每次交易只允许沿着下面的顺序推进，任何一步失败都回到观察：

| 步骤 | 判断问题 | 通过 | 不通过 |
| --- | --- | --- | --- |
| 1 | 周线/日线是否允许当前方向？ | 进入结构识别 | 不交易，只观察 |
| 2 | 操作级别中枢是否清晰？ | 标记 `ZD/ZG/DD/GG` | 不交易，不硬画 |
| 3 | 当前是否处在买卖点区域？ | 进入区间套 | 等待离开、回试或背驰 |
| 4 | 次级别是否同向确认？ | 计算仓位和止损 | 降为试探或放弃 |
| 5 | 入场到失效位是否可承受？ | 下单或挂单 | 缩仓；仍超限则放弃 |
| 6 | 入场后是否仍在原结构内？ | 持有并移动止损 | 触发失效即退出 |

这张表比买卖点名称更优先。一个形态看起来像三买，但如果第 1、4、5 步不通过，就不能按三买执行。

### 3.2 市场状态过滤

| 状态 | 识别方式 | 允许动作 |
| --- | --- | --- |
| 趋势上行 | 价格在日线中枢上方，回试不破，4h 低点抬高 | 二买、三买、持仓、加仓 |
| 中枢震荡 | 价格反复回到 `ZD/ZG` 内，MACD 来回穿越 0轴 | 轻仓高抛低吸或不交易 |
| 趋势下行 | 价格在日线中枢下方，反弹不回中枢，4h 高点降低 | 减仓、空仓、只做极小反弹 |
| 插针失真 | 单根 K 线击穿边界后快速收回，次级别无延续 | 等二次确认，不追涨杀跌 |

BTC 的插针和周末流动性会制造假三买/三卖。只看瞬间刺破或瞬间站回中枢，容易把噪音当成结构完成。

## 4. BTC 版三类买点

### 第一类买点：背驰试探

定义：下跌走势末段价格创新低，但 MACD 绿柱面积或同向下跌长度弱于前一段，提示下跌力量衰竭。

执行：

- 只在日线或 4h 出现清晰下跌结构时考虑。
- 入场前必须有次级别底分型、二次不创新低，或 30m 小级别一买。
- 仓位只做试探，账户风险 0.5%-1%。
- 跌破背驰低点即退出，不摊平。

验证参考：日线第一类买点近似信号 7 个，成功率 85.71%，前向中位收益 16.18%。样本少，只能说明有预警价值，不能当作稳定胜率。

### 第二类买点：确认加仓

定义：第一类买点后出现反弹，第一次回调不跌破第一类买点低点。

执行：

- 一买后必须先看到向上笔或小级别上涨结构。
- 首次回调低点高于一买低点，且回调力度弱。
- 重新突破回调前小高点时加仓。
- 若回调跌破一买低点，前一套买点假设作废。

验证参考：4h 第二类买点近似信号 9 个，成功率 66.67%，前向中位收益 9.77%。它比一买更适合作为实盘加仓点。

### 第三类买点：趋势确认

定义：向上离开中枢后，第一次回试不跌回中枢上沿 `ZG`。

执行：

- 中枢必须清楚，`ZG` 不能靠主观移动。
- 离开中枢时最好伴随 4h MACD 回到 0轴上方或红柱扩张。
- 回试不破 `ZG` 后，在 30m 出现底分型/小级别二买时入场。
- 买入后跌回中枢，立即降仓；有效跌破 `ZD`，退出趋势假设。

验证参考：4h 第三类买点近似信号 7 个，成功率 28.57%，前向中位收益 4.63%。三买是趋势确认，不是无条件重仓信号。

## 5. BTC 版三类卖点

第一类卖点：上涨走势末段价格创新高，但 MACD 红柱面积或同向上涨长度弱于前一段。动作是减仓和上移止损，不必一次性猜顶。

第二类卖点：一卖后下跌，再次反弹不创新高。若同时跌破 4h 中枢下沿或日线短均线，退出趋势仓。

第三类卖点：向下离开中枢后，第一次回抽不升破 `ZD`。它用于确认下跌结构和保护利润，不建议在现货里把它等同于开空信号。

验证参考：日线第一类卖点近似信号 5 个，成功率 80.00%；4h 第二类卖点近似信号 11 个，成功率 36.36%；4h 第三类卖点近似信号 7 个，成功率 28.57%。

## 6. 信号分级

| 等级 | 条件 | 动作 |
| --- | --- | --- |
| A | 周线不弱，日线买点成立，4h 同向确认，30m 给出可执行触发，潜在空间至少 1.5R | 标准仓位，允许分批加仓 |
| B | 日线买点成立但 4h 只部分确认，或空间刚够 1.2R-1.5R | 半仓以下，先试后加 |
| C | 只有 4h/30m 信号，日线环境不配合 | 观察或极小短线 |
| 禁止 | 周线和日线同时弱，或止损距离过大，或刚跌回中枢 | 不交易 |

30m 买点类信号本次共有 282 个，加权成功率 55.32%。它说明 30m 有执行价值，但不能脱离 1d/4h 环境单独重仓。

### 6.1 信号评分细则

为了避免“看起来都像机会”，每个候选交易先按 10 分制评分：

| 项目 | 分值 | 给分条件 |
| --- | --- | --- |
| 大级别环境 | 2 | 周线不弱，日线不在中枢下方持续走弱 |
| 中枢清晰度 | 2 | `ZD/ZG` 可客观标注，边界没有频繁被打穿 |
| 买卖点质量 | 2 | 一买有背驰，二买不破一买，三买/三卖不回中枢 |
| 区间套确认 | 2 | 4h/30m 出现同向分型、背驰或小级别买卖点 |
| 风险收益比 | 2 | 第一目标至少 1.5R，且止损距离不迫使重仓冒险 |

| 总分 | 处理 |
| --- | --- |
| 8-10 | A 级，允许标准仓位 |
| 6-7 | B 级，只能半仓以下 |
| 4-5 | C 级，只观察或极小短线 |
| 0-3 | 禁止交易 |

评分必须在下单前完成，不能入场后再为持仓补理由。

## 7. 仓位与风险

| 场景 | 仓位上限 | 失效位 |
| --- | --- | --- |
| 第一类买点试探 | 20%-30% | 背驰低点下方 |
| 第二类买点确认 | 40%-60% | 二买低点下方 |
| 第三类买点趋势确认 | 30%-60% | 跌回中枢或跌破回试低点 |
| A 级共振信号 | 60%-80% | 按结构止损，账户风险不超过 2% |
| 周线弱势下 4h/30m 买点 | 0%-20% | 触发失败立即退出 |

单笔账户风险控制在 0.5%-2%。杠杆交易先用止损距离反推仓位，强平价必须远离结构止损；如果止损距离过大，正确动作是降低仓位或放弃，而不是把止损移近。

### 7.1 退出和移动止损

| 情况 | 动作 |
| --- | --- |
| 入场后跌回买入所依据的中枢 | 先降仓，等待下一根操作级别 K 线确认 |
| 有效跌破一买/二买低点或三买回试低点 | 退出，不摊平 |
| 达到 1R | 可把止损上移到结构低点或入场价附近 |
| 达到 1.5R-2R | 至少兑现部分利润，剩余仓位跟随 4h 结构 |
| 出现日线一卖或 4h 二卖 | 降低趋势仓，保留底仓需有明确周线理由 |
| 价格重新进入大级别中枢并失去方向 | 停止加仓，把策略切换为震荡或空仓 |

止盈不要求卖在最高点。缠论在这里的作用是识别结构是否仍然成立，而不是预测极限价格。

| 字段 | 记录 |

| 操作级别 | 1d / 4h / 30m |
| 大级别环境 | 1w、1d 是否在中枢上方/下方/内部 |
| 当前中枢 | `ZD`、`ZG`、`DD`、`GG` |
| 买卖点类型 | 一买/二买/三买/一卖/二卖/三卖 |
| 区间套确认 | 4h 或 30m 是否同向 |
| 背驰证据 | MACD 面积、绝对长度、量能是否支持 |
| 入场触发 | 分型确认、突破小高点、回试不破 |
| 止损 | 结构低点/高点、中枢边界、回试低点 |
| 目标 | 前高/前低、上方/下方中枢、1.5R |
| 信号等级 | A / B / C / 禁止 |

### 8.1 复盘记录模板

| 字段 | 记录 |
| --- | --- |
| 结果 | 盈利 / 亏损 / 保本 |
| 实际 R 倍数 | 实际盈亏除以初始风险 |
| 最大有利波动 | 是否达到报告中的统计阈值 |
| 最大不利波动 | 是否接近或触发失效位 |
| 执行偏差 | 是否提前入场、追价、移走止损、混级别 |
| 结构复盘 | 原中枢和买卖点是否画错 |
| 可复用结论 | 保留规则 / 修改规则 / 删除规则 |

连续 20 笔交易后再评估规则，不用单笔盈亏否定或神化一个买卖点。

## 9. 禁用清单

- 不在没有中枢边界时硬找第三类买卖点。
- 不把中枢内部的普通背离当作趋势背驰。
- 不因为“三买重要”就忽略大级别弱势。
- 不用 5m 信号推翻日线计划。
- 不在跌回中枢后继续假装趋势已经确认。
- 不使用固定概率、固定仓位和“必参与”口号。
- 不把最大有利波动统计当成实盘收益。
- 不在同一笔交易中途切换操作级别，例如按日线买入、按 5m 恐慌卖出。
- 不在止损过远时用更小级别止损伪装低风险。

## 10. 每日检查清单

- [ ] 周线环境是否允许做多？
- [ ] 日线中枢 `ZD/ZG` 是否清晰？
- [ ] 当前处于中枢内、离开、回试，还是失效回中枢？
- [ ] 日线或 4h 是否出现一买/二买/三买？
- [ ] 4h/30m 是否完成区间套确认？
- [ ] 背驰比较是否只发生在中枢外的同向段？
- [ ] 入场到止损的距离是否允许建仓？
- [ ] 信号等级是否达到 B 级以上？
- [ ] 若判断错误，账户亏损是否在 2% 内？

### 10.1 周度复盘清单

- [ ] 本周是否只交易了 B 级以上信号？
- [ ] 是否出现混级别操作？
- [ ] 是否有未按失效位退出的交易？
- [ ] 哪类买卖点贡献了主要盈利或亏损？
- [ ] 亏损交易是结构识别错误，还是执行纪律错误？
- [ ] 下周是否需要降低操作级别、减少仓位，或停止交易？

## 11. 一句话版本

用周线定环境，用日线画主中枢，用 4h 判断买卖点是否被承接，用 30m/5m 执行；背驰只做预警，二买和三买做确认，所有交易先写失效位，再谈仓位。

# BTC 新威科夫操盘法：验证后可复用交易方法论

> 数据基础：Binance `BTCUSDT` 最近两年 K 线，验证区间 2024-06-28 至 2026-06-27。若最后一根日线/周线尚未收盘，只作为最新覆盖，不作为独立触发信号。
> 适用对象：BTCUSDT 日线波段交易；周线定背景，日线定计划，4小时/1小时做执行细节。本文不是投资建议。

## 1. 方法论定位

这份方法论只保留《新威科夫操盘法》中能被 BTC K 线复验、且能写成流程的内容：供求关系、努力与结果、因果准备、三高三低、吸筹/派发流程、Spring/TSO、SOS/JOC、LPS、UT/SOW、破冰/测冰、持仓秩序和危机管理。

所有“CM/主力正在做什么”的表述，在执行时统一改写成市场自身行为：放量推进、放量无进展、缩量测试、跌破收回、突破失败、低量回测、承接、卖压、三高、三低、破冰。

## 2. 总流程

每天只按固定顺序回答七个问题：

1. 周线背景：多头、中性还是空头？
2. 日线位置：趋势中、区间内、吸筹右手边、派发右手边，还是末端风险区？
3. 供求状态：需求推进、供应释放、供应枯竭，还是需求耗尽？
4. 行为标签：SC/ST、Spring/TSO、SOS/JOC、LPS、UT/SOW、破冰/测冰，还是噪声？
5. 确认质量：是否有三高/三低、放量跟随、缩量回测、跌回区间或跌破结构？
6. 风险位置：止损在何处，第一目标是否至少 1.5R？
7. 危机管理：进场后 1-5 根哪些行为触发退出？

优先级固定：风险与退出 > 周线背景 > 日线位置 > 价量确认 > 4小时执行。低级别信号不能推翻高级别风险。

### 2.1 三段式压缩版

实盘中把七个问题压缩成三段，避免看盘时越看越散：

| 阶段 | 只回答一个问题 | 允许动作 |
| --- | --- | --- |
| 观察 | 背景和位置是否值得做？ | 标记候选，不下单 |
| 计划 | 是否出现 A/B 级行为且风险可量化？ | 写入场、止损、目标、危机条件 |
| 执行 | 市场是否按计划确认或否定？ | 触发、持仓、减仓、退出 |

没有从“观察”进入“计划”的信号，不允许临盘变成交易；没有写清危机条件的计划，不允许执行。

## 3. 数据定义与证据等级

| 概念 | BTC 执行定义 | 用法 |
| --- | --- | --- |
| 高量 | 当前量 >= 20根均量的1.25-1.50倍 | 识别推动、异常、卖压或承接 |
| 低量 | 当前量 <= 20根均量的0.80-0.95倍 | 二次测试、LPS、回测、缩量回调 |
| 宽幅 | 当前振幅 >= 20根均振幅的1.05-1.25倍 | 识别推进、SC/BC、SOW/破冰 |
| 小实体 | 实体 <= 全振幅25% | 努力无结果、吸收或分歧 |
| 强实体 | 实体 >= 全振幅40%-55% | 推进、突破或跌破确认 |
| 三高 | 高点、低点、收盘均高于上一根 | 上涨秩序候选 |
| 三低 | 低点、高点、收盘均低于上一根 | 下跌秩序候选 |

日线样本中，放量宽幅实体顺向命中率 51.85%，放量窄幅/小实体作为反向警报的命中率 57.14%；三高且量不低的顺向命中率 57.47%，三低且量不低的反向命中率 52.75%。结论：秩序语言可用，但必须结合背景和位置。

| 等级 | 条件 | 用法 |
| --- | --- | --- |
| A级 | 多周期一致、样本不少、结果与原理同向 | 可进入评分与仓位 |
| B级 | 样本有限但和原理同向 | 只做候选，必须等确认 |
| C级 | 样本稀疏或结果不稳定 | 只做提醒，不作为入场理由 |
| 不可验证 | 主力身份、消息链路、媒体配合 | 只能作为叙事，不能作为触发 |

样本可靠性按更保守的规则解释：

| 样本规模 | 解释方式 |
| --- | --- |
| 少于10个 | 只记录现象，不引用命中率做决策 |
| 10-30个 | 可作为 B/C 级候选，必须等二次确认 |
| 30个以上 | 可进入评分，但仍需结合背景、位置和风险收益 |
| 多周期同向 | 证据升级 |
| 周线样本很少 | 只做背景提示，不做精确统计结论 |

## 4. 背景判断

| 周线状态 | 判断 | 日线动作 |
| --- | --- | --- |
| 多头 | 周收盘在20周均线上方，未连续破结构 | 允许标准多头计划 |
| 中性 | 周线横盘，日线在大区间内 | 只做边缘、确认和小仓 |
| 空头 | 周收盘在20周均线下方，或放量破结构 | 禁止因单根 Spring/长下影重仓抄底 |

日线 JOC 与周线多头背景同向时，本次样本命中率 64.29%，平均结果 +0.36%，样本 14 个。周线是过滤器，不是装饰。

## 5. 吸筹流程的可交易版本

吸筹只能写成可观察行为：

1. 先有下跌末端或区间下沿。
2. 出现 SC 或放量止跌，只标记供应释放/承接候选。
3. 等 ST：回到前低附近时，成交量明显低于 SC，振幅缩小，不能有效跌破。
4. 等 Spring/TSO 或 SOS/JOC：跌破收回，或放量离开区间。
5. 优先在二次测试、低量回测、4小时转强、或日线重新收上关键位后执行。

日线 SC 后缩量二次测试命中率 62.50%；单根恐慌 K 线不准直接抄底。

## 6. Spring 与 TSO

| 类型 | BTC 定义 | 动作 |
| --- | --- | --- |
| 低量 Spring | 小幅跌破20根低点后收回，成交量低于或接近均量 | 候选，等4小时确认或下一根不破低点 |
| 放量 Spring | 跌破收回但成交量高 | 只观察，等二次测试或连续上行确认 |
| TSO | 深度跌破55根低点后收回 | C级候选，必须等缩量二次测试 |
| 确认 Spring | Spring 后1-3根突破信号 K 高点且不破低点 | 可进入交易计划 |
| 失败 Spring | Spring 后1-3根跌破信号 K 低点 | 退出，禁止补仓 |

日线 Spring 后 1-3 根向上确认命中率 70.00%；日线 TSO 后缩量二次测试样本 0 个，必须按 C 级使用。

## 7. SOS/JOC 与 LPS

JOC/SOS 是最核心的多头触发，但标准仓位优先等回测或小周期确认。

JOC/SOS 条件：

1. 日线有明确20-55根震荡区或前高压力。
2. 收盘突破区间上沿或20根高点。
3. 成交量 >= 20根均量1.25倍。
4. 实体不小，收盘靠近高位。
5. 周线不明显逆向。

LPS 条件：

1. 周线多头，日线在50/200日均线之上。
2. 已出现过 SOS/JOC 或趋势上行。
3. 回调阶段成交量下降，振幅收缩，没有放量跌破结构。
4. 4小时重新转强，或日线重新收上回调高点。

日线 JOC/SOS 放量突破命中率 54.55%；日线 LPS/趋势中缩量最后支撑命中率 68.75%。二者都必须配合止损距离和 1-5 根复核。

## 8. 派发、UT/SOW 与破冰

| 信号 | 执行定义 | 动作 |
| --- | --- | --- |
| SOT | 上涨推进距离缩短，量能不再扩张 | 收紧止损，禁止追涨 |
| UT | 上冲20根高点但收盘跌回 | 多头减仓或禁止新增 |
| SOW/破冰 | 放量强实体跌破20根低点 | 退出，等待反弹弱再评估 |
| 测冰 | 跌破后低量弱反弹，无法收回原支撑 | 可作做空候选，但需额外验证 |
| JOC失败 | 突破后1-5根放量跌回区间 | 危机管理退出 |

UT/SOW/破冰优先服务于退出和降仓，不机械做空。BTC 做空还需要资金费率、滑点、杠杆、强平价和止损距离约束。

## 9. 三类允许入场

### A. JOC 低量回测

1. 周线多头或中性，不是明确空头。
2. 日线放量 JOC/SOS 突破区间。
3. 回测原阻力时成交量 <= 20根均量0.90倍，收盘不跌回区间。
4. 4小时重新转强，或日线下一根收回回测 K 高点。
5. 止损放在回测低点或 JOC K 线低点下方。

### B. Spring/TSO 确认

1. 日线位于上升趋势回调、区间下沿，或疑似吸筹右手边。
2. 跌破前低后收回，不能是周线空头中的孤立诱多。
3. 放量时必须等二次测试缩量；低量时仍需下一根或4小时确认。
4. 入场后 1-5 根必须出现向上跟随，否则退出。
5. 止损放在 Spring/TSO 低点外侧。

### C. LPS 缩量回调延续

1. 周线多头，日线趋势未破。
2. 回调缩量、窄幅，没有放量三低或破冰。
3. 4小时放量收回短压，或日线重新收阳突破回调高点。
4. 止损放在回调低点下方。

其他信号一律降级为观察：单根 SC、长下影、单根高量小实体、缩量突破、周线空头中的 Spring、孤立 SOT。

### 9.1 入场状态机

| 状态 | 进入条件 | 下一步 |
| --- | --- | --- |
| 空仓 | 无 A/B 级信号，或风险收益不足 | 等待 |
| 候选 | 出现 SC/ST、Spring、JOC、LPS、UT/SOW 之一 | 标注关键位和失效位 |
| 计划 | 周线不逆向，信号评分 >= 6，止损和目标清楚 | 等触发 |
| 触发 | 价格到达计划入场位，4小时不逆向 | 建仓 |
| 持仓 | 入场依据仍成立 | 跟踪止损和三高/三低 |
| 危机 | 1-5根内出现否定行为 | 减仓或退出 |
| 复盘 | 交易结束 | 记录是否遵守计划 |

任何状态都可以回到空仓；不能从“候选”直接跳到“持仓”。

### 9.2 执行周期分工

| 周期 | 职责 | 禁止事项 |
| --- | --- | --- |
| 周线 | 决定是否允许标准仓位 | 不用单根周线追涨杀跌 |
| 日线 | 生成交易计划、止损和目标 | 不因1小时信号改变日线方向 |
| 4小时 | 优化入场、观察回测质量 | 不推翻周线空头或日线破冰 |
| 1小时 | 只用于细化触发和减少滑点 | 不独立产生波段交易理由 |

## 10. 持仓与退出

持仓只看秩序是否仍成立：

| 持仓依据 | 正常 | 风险 |
| --- | --- | --- |
| 上涨趋势 | 上攻量增价进，回调量缩幅缩 | 回调量增幅增、巨量不涨、三低 |
| JOC | 不跌回区间，回测缩量 | 放量跌回、长上影、连续无需求 |
| Spring/TSO | 不破低点，低点抬高 | 1-3根跌破信号低点 |
| LPS | 缩量回调后恢复三高 | 放量跌破回调低点 |

出现 UT、SOW/破冰、JOC 失败、放量宽幅阴线、跌破结构止损时，先退出或降仓，再等待新确认。

## 11. 危机管理

进场后 1-5 根必须复核。日线危机样本中，JOC 后1-5根放量跌回区间的反向命中率 40.00%，Spring 后1-3根跌破低点的反向命中率 72.73%，上涨后放量宽幅阴线的反向命中率 58.33%。

危机处理顺序：

1. 缩紧止损到结构低点/高点外。
2. 若触发否定条件，先退出，不解释成洗盘。
3. 退出后至少等一根日线重新确认。
4. 禁止用补仓摊低成本修正错误入场。

## 12. 仓位与评分

| 场景 | 仓位 |
| --- | --- |
| 周线多头 + 日线 JOC + 低量回测成功 | 50%-100%计划仓位 |
| Spring 确认且止损很近 | 20%-50%计划仓位 |
| TSO 后二次测试成功 | 10%-30%试探仓 |
| LPS 缩量回调延续 | 30%-70%计划仓位 |
| 周线中性 | 标准仓位减半 |
| 周线空头、单根 SC/长下影、未确认 Spring | 默认 0%，只观察 |

单笔计划亏损控制在账户权益的 0.5%-1.5%。先定止损，再用止损距离反推数量；止损距离超过日线 ATR 的 1.5-2.0 倍时，默认等待或放弃。

成本过滤必须在下单前完成：

| 项目 | 要求 |
| --- | --- |
| 手续费+滑点 | 预估成本不得吃掉第一目标收益的20%以上 |
| 止损距离 | 小于日线 ATR 的1.5-2.0倍，否则等回测 |
| 目标距离 | 第一目标至少1.5R，低于1.5R不交易 |
| 流动性 | 极端插针或价差异常时只减仓不加仓 |
| 杠杆 | 先按现货或低杠杆验证，不能用杠杆弥补低胜率 |

评分低于6分不交易：

| 项目 | 分值 | 条件 |
| --- | --- | --- |
| 周线背景 | 2 | 同向或至少不逆向 |
| 日线位置 | 2 | 右手边、突破边缘、趋势健康回调 |
| 价量行为 | 2 | JOC、确认 Spring/TSO、LPS 之一 |
| 确认质量 | 2 | 放量跟随、回测缩量、不跌回区间 |
| 风险收益 | 1 | 第一目标至少1.5R |
| 执行周期 | 1 | 4小时同向，未出现失败信号 |

出现 UT、SOW/破冰、JOC 失败或跌破结构时，评分自动失效。

## 13. 执行模板

| 字段 | 内容 |
| --- | --- |
| 周线背景 | 多头/中性/空头 |
| 日线阶段 | 吸筹右手边、趋势回调、JOC突破、派发警报等 |
| 理论依据 | 供应枯竭、需求确认、低量回测、突破失败等 |
| 行为标签 | SC/ST、Spring/TSO、SOS/JOC、LPS、UT/SOW、破冰/测冰 |
| 价量证据 | 成交量倍数、实体比例、收盘位置、三高/三低 |
| 入场触发 | JOC回测、Spring确认、LPS回调转强 |
| 止损位置 | Spring低点、回测低点、JOC低点或结构低点 |
| 计划亏损 | 账户权益百分比 |
| 第一目标 | 前高、区间等幅、2R或上方阻力 |
| 危机管理 | 1-5根内哪些行为触发退出 |
| 复盘结果 | 信号评分、是否等确认、是否按止损执行 |

### 13.1 单笔交易验收

下单前必须能完整写出这五句话：

1. 我交易的是哪一类入场：JOC 回测、Spring/TSO 确认，还是 LPS？
2. 如果判断正确，价格应在几根 K 线内出现什么行为？
3. 如果判断错误，哪一个价格或量价行为会证明我错了？
4. 我的第一目标在哪里，是否至少 1.5R？
5. 这笔交易失败时，账户亏损是否仍在计划内？

五句话写不清，说明交易还停留在“观察”状态。

## 14. 禁用规则

- 不用“主力正在吸筹/派发”作为下单理由，只记录可见行为。
- 不在单根 SC、BC、长下影、巨量小实体出现时直接入场。
- 不把放量 Spring 或 TSO 当成低风险买点，必须二次确认。
- 不追周线空头中的日线缩量突破。
- 不在 JOC 后放量跌回区间时加仓。
- 不把 UT/SOW/破冰机械等同于做空信号。
- 不用新闻解释亏损交易，除非事前已写入计划。

暂停新开仓条件：

- 连续3笔亏损，且至少2笔低于6分。
- 最近5笔交易中有2笔违反“先退出再等确认”。
- 月度已实现亏损达到预设上限。
- 简化规则连续失效，仍想提高仓位。

## 15. 复盘与迭代

每20笔交易或每月复盘一次：

1. 亏损是否来自低于6分的信号？
2. 是否在 SC、长下影、放量 Spring、TSO 上提前抄底？
3. 是否等到了 JOC 后低量回测或 LPS？
4. 危机管理是否在 1-5 根内执行？
5. 盈利单是否遵守了 UT/SOW/破冰或放量跌破结构的退出？

本次简化“日线新威科夫组合规则”总收益 12.68%、最大回撤 -13.00%、在场时间 20.96%；同期买入持有总收益 0.30%、最大回撤 -52.03%。这说明本方法应定位为人工决策框架，不应包装成自动盈利系统。

实盘启用前的最低验收标准：

| 指标 | 标准 |
| --- | --- |
| 纸面/小仓样本 | 至少30笔，且覆盖 JOC、Spring/TSO、LPS 三类场景 |
| 计划遵守率 | 至少90%的交易有完整模板和事前止损 |
| 违规交易占比 | 低于10%，且不能出现未设止损重仓 |
| 最大实际回撤 | 不超过预设月度亏损上限 |
| 期望值 | 扣除手续费和滑点后仍为正，或亏损可明确归因并修正 |
| 危机管理 | JOC失败、Spring失败时能先退出再等待 |

## 16. 一句话版本

先分背景，再等右手边，只交易 JOC 回测、确认 Spring/TSO 和 LPS；所有单子先写止损和危机条件，进场后 1-5 根不被市场确认就退出。

# BTC 缠论验证后可复用交易方法论

> 标的：BTCUSDT  
> 数据验证区间：2024-06-27 至 2026-06-27  
> 适用对象：BTC 现货或低杠杆趋势/波段交易  
> 核心原则：只交易已经生长出来的结构，不预测未完成的走势。

## 1. 适用结论

缠论在 BTC 上可复用的部分，是“结构识别 + 买卖点确认 + 多级别联立 + 风险纪律”。不可复用为机械预测术，也不能把任何单一背驰、分型或中枢突破当成必胜信号。

本方法论保留以下内容：

- K线包含处理、顶底分型、笔、中枢、三类买卖点。
- 背驰和盘整背驰作为转折预警。
- 第三类买卖点作为右侧结构确认和失效管理依据。
- 区间套和多级别联立作为提高信号质量的过滤器。
- MACD 0轴作为弱势过滤器。

以下内容只作为风险提示，不作为硬规则：

- 第二个中枢后一定背驰或固定概率背驰。
- 背驰后必然出现当前级别大反转。
- 通过频繁短差把 BTC 持仓做到 0 成本。

## 2. 周期配置

| 角色 | 推荐周期 | 用途 |
| --- | --- | --- |
| 大级别 | 1w | 判断主趋势、是否处在大级别中枢上/下方 |
| 操作级别 | 1d | 决定主要买卖点和仓位 |
| 次级别 | 4h | 验证日线买卖点、寻找更精确入场 |
| 执行级别 | 1h | 优化触发价和止损，不反向推翻日线结构 |

日线是默认操作级别。只有资金期限短、能盯盘且严格止损时，才把 4h 作为操作级别。

## 3. 结构识别流程

每次分析按固定顺序执行：

1. 对 K 线做包含处理，只保留高低点区间。
2. 识别顶分型和底分型。
3. 用相邻顶底连接成笔，过滤幅度过小的噪音笔。
4. 用连续三笔重叠识别中枢，记录 `ZD/ZG/DD/GG`。
5. 判断当前位置属于中枢内震荡、离开中枢、回试中枢，还是回到中枢内失效。
6. 再看 MACD 面积、0轴位置和多级别确认。

先有结构，后有指标。MACD 只辅助判断力度，不能替代中枢和买卖点。

## 4. 三类买点规则

### 第一类买点

定义：下跌趋势末段价格创新低，但 MACD 绿柱面积或力度弱于上一段同向下跌，形成底背驰。

执行条件：

- 日线或 4h 已有清晰下跌结构，且不是单纯中枢内乱震。
- 价格创新低，但 MACD 面积明显缩小。
- 次级别出现底分型确认，或 1h/4h 不再创新低。
- 止损放在背驰低点下方，跌破即承认背驰失败。

日线样本中，第一类买点近似信号 7 个，成功率 85.71%，未来窗口中位收益 16.18%。这个结果说明它有预警价值，但样本偏少，不能把该比例当成稳定胜率；执行上只允许小仓试探，等第二类买点或 4h 结构确认后再加仓。

### 第二类买点

定义：第一类买点后出现向上笔，随后第一次回调不跌破第一类买点低点。

执行条件：

- 第一类买点已引发可见反弹。
- 首次回调缩量或 MACD 不再明显走弱。
- 回调低点高于第一类买点低点。
- 再次向上突破回调前小高点时加仓。

第二类买点是 BTC 更稳健的左侧转右侧位置。若第二类买点跌破第一类买点低点，前面的背驰假设作废。

### 第三类买点

定义：次级别走势向上离开中枢后，第一次回试低点不跌破中枢上沿 `ZG`。

执行条件：

- 中枢已明确，`ZD/ZG` 不含糊。
- 向上离开中枢的力度足够，最好伴随 MACD 回到 0轴上方。
- 第一次回试不跌回 `ZG`，且 4h 或 1h 出现底分型/小级别买点。
- 买入后若收回中枢内，立即降仓或退出。

本次样本中，严格日线第三类买点近似信号 0 个；4h 第三类买点近似信号 7 个，成功率 28.57%，未来窗口中位收益 4.63%。因此第三类买点在 BTC 上不能单独作为重仓理由，更适合作为“日线环境已经偏多时，4h 回试不破”的结构确认；若风险收益比不足 1.5R，宁可放弃。

## 5. 三类卖点规则

第一类卖点：上涨趋势末段价格创新高，但 MACD 红柱面积或力度弱于上一段上涨。先减仓，不等确认时可保留底仓。

第二类卖点：第一类卖点后首次反弹不创新高。若同时跌破日线 5日/10日均线或 4h 中枢下沿，退出趋势仓。

第三类卖点：向下离开中枢后，第一次回抽高点不升破 `ZD`。本次样本中，严格日线第三类卖点近似信号 0 个；4h 第三类卖点近似信号 7 个，成功率 28.57%。它更适合用来降低趋势仓和定义反弹失效，而不是单独开反向仓。

## 6. 多级别联立

交易前必须回答三个问题：

| 问题 | 通过条件 |
| --- | --- |
| 周线是否允许做多 | 价格在周线中枢上方、或周线 MACD 不在 0轴下持续走弱 |
| 日线买点是否成立 | 出现第一/第二/第三类买点之一，且止损可控 |
| 4h 是否确认 | 4h 出现同向底分型、小级别背驰或第三类买点 |

日线买点没有 4h 确认时，最多试探仓。日线卖点被 4h 确认时，先减仓再讨论是否会反包。

## 7. 信号分级

| 等级 | 条件 | 动作 |
| --- | --- | --- |
| A | 周线不弱，日线买点成立，4h 同向确认，止损内 1.5R 以上空间 | 标准仓位，允许分批加仓 |
| B | 日线买点成立，但 4h 只部分确认，或空间不足 1.5R | 半仓以下，先试后加 |
| C | 只有 4h/1h 信号，日线环境不配合 | 观察或极小短线仓 |
| 禁止 | 周线、日线同时弱势，或信号失效位过远 | 不交易 |

信号分级优先于买点名称。一个第三类买点若没有大级别环境支持，只能是 C 级；一个第一类买点若随后出现第二类确认和 4h 共振，可以升级为 A/B 级。

## 8. 仓位与止损

| 信号 | 仓位 |
| --- | --- |
| 第一类买点 | 20%-40%，只做试探 |
| 第二类买点 | 40%-70%，确认后加仓 |
| 第三类买点 | 30%-60%；只有达到 A 级时才允许继续加仓 |
| 周线弱势下的 4h 买点 | 0%-20%，只允许短线 |
| 第三类卖点或日线中枢下破 | 降至 0%-30% |

单笔账户风险控制在 0.5%-2%。杠杆交易必须以结构止损计算真实亏损，强平价不能接近结构止损。

## 9. 入场模板

| 字段 | 记录 |
| --- | --- |
| 操作级别 | 1d/4h |
| 当前中枢 | `ZD`、`ZG`、`DD`、`GG` |
| 买点类型 | 第一类/第二类/第三类 |
| 多级别状态 | 周线、日线、4h 是否同向 |
| MACD | 0轴上/下、面积是否背驰 |
| 入场触发 | 分型确认、突破小高点、回试不破 |
| 止损 | 背驰低点、二买低点或中枢上沿失守 |
| 第一目标 | 前高、上方中枢、1.5R |
| 失效条件 | 跌回中枢、跌破低点、MACD 再次走弱 |
| 信号等级 | A/B/C/禁止 |

## 10. 禁用规则

- 不在没有中枢定义的地方硬找第三类买点。
- 不在盘整里把普通 MACD 背离叫趋势背驰。
- 不因为“跌得多”买入，必须有分型、背驰或回试确认。
- 不因为“突破中枢”立即追入，必须等待第一次回试。
- 不在周线和日线同时弱势时重仓做 1h 反弹。
- 不把任何固定概率、口号或历史案例当成 BTC 当前交易依据。
- 不把“信号后最大有利波动”理解为实盘可获得收益，入场前必须先计算止损和滑点。

## 11. 每日检查清单

- [ ] 周线处于中枢上方、下方，还是中枢内？
- [ ] 日线当前中枢 `ZD/ZG` 是否清晰？
- [ ] 当前是中枢震荡、离开、回试，还是失效回中枢？
- [ ] 是否出现第一/第二/第三类买卖点？
- [ ] 信号等级是 A、B、C，还是禁止？
- [ ] MACD 是否支持力度判断？是否在 0轴下方走弱？
- [ ] 4h 是否与日线同向确认？
- [ ] 入场到止损的距离是否允许仓位介入？
- [ ] 若判断错误，账户亏损是否在 2% 内？

## 12. 一句话版本

用周线定环境，用日线画中枢和三类买卖点，用 4h 确认背驰或回试；先给信号分级，再决定仓位，只在结构完成且止损可控时交易，跌回中枢或跌破结构低点就退出。

# BTC 海龟交易法则：验证后可复用交易方法论

> 数据基础：Binance `BTCUSDT` 最近两年 K 线，验证区间 2024-06-27 至 2026-06-27。
> 适用对象：BTCUSDT 趋势跟踪交易；不构成投资建议。

## 1. 方法论定位

这套方法只保留《海龟交易法则》中能在 BTC K 线上验证、并能转化为可执行交易流程的内容：突破入场、N 值风险单位、2N 止损、0.5N 加仓、反向通道退出、趋势过滤和复盘指标。

原书的多期货市场分散化、合约滚动、场内限价单战术和相关市场头寸限制，不直接进入 BTC 单品种规则；它们被改写为组合扩展和执行纪律。

本次验证后的使用边界：

- 日线原版系统在 2024-06-27 至 2026-06-27 的 BTCUSDT 样本中没有直接跑出正收益，因此不能把 20/55 日突破理解为“照搬即可盈利”的参数。
- 4小时系统表现更好，但交易频率、滑点、手续费和资金费率会显著影响实盘结果。
- 最值得复用的是“完整系统 + N值仓位 + 2N止损 + 反向通道退出 + 复盘纪律”，而不是某个单一突破参数。
- BTC 单品种缺少原书最重要的多市场分散化，默认仓位必须比原版更保守。

提炼后的核心取舍：

| 原书内容 | BTC保留方式 | 使用结论 |
| --- | --- | --- |
| 20/55突破 | 保留为趋势触发器 | 可用，但必须经过趋势过滤和仓位折扣 |
| N值仓位 | 完整保留 | 这是最重要的可复用模块 |
| 2N止损 | 完整保留但降低单位风险 | BTC波动大，2N距离常常不小 |
| 0.5N加仓 | 保留但减少默认满仓次数 | 单品种常态2-3单位，强趋势才4单位 |
| 多市场分散化 | 降级为扩展模块 | 只做BTC时不能享受原书分散化优势 |
| 盘中突破即成交 | 改为收盘确认或执行周期确认 | 降低假突破，但会牺牲部分入场速度 |
| 机械执行纪律 | 完整保留 | 这是方法能否复用的关键 |

## 2. 第一原则：只交易完整系统

海龟法不是“看到突破就买”，而是一套完整系统。每一笔 BTC 交易必须同时写清：

1. 市场：只交易流动性足够的 BTCUSDT 主流交易场所。
2. 入场：系统1的 20 单位突破，或系统2的 55 单位突破。
3. 止损：入场即设置 2N 失效点。
4. 退出：系统1用 10 单位反向突破，系统2用 20 单位反向突破。
5. 仓位：每个单位按 N 值折算，不按感觉买币。
6. 加仓：只在盈利方向每推进 0.5N 后加仓，最多 4 单位。
7. 复盘：每笔交易统计 R 倍数、是否按规则执行、是否跳过信号。

本次样本中，买入持有总收益 -0.13%、最大回撤 -52.03%；日线系统1总收益 -18.54%、最大回撤 -49.90%；日线系统2总收益 -10.56%、最大回撤 -31.79%。结论是：海龟法可复用的核心不是静态持有，而是用规则把亏损封顶、把趋势利润放开。

## 3. 周期职责分工

| 周期 | 职责 | 规则 | 用法 |
| --- | --- | --- | --- |
| 周线 | 大趋势背景 | 55周突破、20周退出、50/300趋势过滤 | 决定是否允许大仓位趋势跟踪 |
| 日线 | 主交易周期 | 20/55日突破、10/20日退出、N值仓位 | 生成主要交易计划 |
| 4小时 | 执行周期 | 20/55根突破、10/20根退出 | 优化入场，不能逆日线主方向重仓 |
| 1小时 | 风险观察 | N值变化、假突破、波动放大 | 只用于执行微调和风控，不改变主系统 |

执行分层：

| 模式 | 适用账户 | 交易周期 | 仓位原则 | 说明 |
| --- | --- | --- | --- | --- |
| 保守模式 | 现货、低频、纪律验证期 | 日线信号 + 4小时执行 | 0.25%-0.50%单位风险，最多2单位 | 优先学会按规则亏小钱 |
| 标准模式 | 已有复盘记录，能接受连续亏损 | 日线过滤 + 4小时系统2 | 0.50%-1.00%单位风险，最多3单位 | 本次样本中更接近可执行候选 |
| 激进模式 | 合约、低杠杆、成熟风控 | 4小时系统1/2并行 | 单位风险不超过0.50%，最多4单位 | 只在周线/日线/4小时同向时使用 |

优先采用“日线定方向，4小时找触发，日线管理总风险”的组合。日线原版参数更适合作为大级别过滤和复盘基准，不宜直接作为满仓交易系统。

## 4. 市场状态过滤

BTC 单品种不具备原书的多市场分散化，所以必须先过滤市场状态。

允许交易趋势仓的环境：

- 日线价格在 300 日均线上方，且 50 日均线高于 300 日均线：优先做多突破。
- 日线价格在 300 日均线下方，且 50 日均线低于 300 日均线：只允许做空系统或空仓观察。
- 周线和日线方向一致：允许标准单位风险。
- 周线和日线冲突：只允许半单位或放弃。

本次样本中，日线 20 日突破 E70 为 1.14，说明突破后的中期有利变动大于不利变动；但过滤器不能保证每个两年窗口收益最高，它的主要作用是减少逆势信号和心理噪声。

交易状态机：

| 状态 | 进入条件 | 允许动作 | 退出条件 |
| --- | --- | --- | --- |
| 观察 | 无突破，或趋势过滤不支持 | 记录20/55高低点、N值和资金费率 | 趋势过滤转为支持 |
| 准备 | 趋势过滤支持，价格接近20/55突破位 | 计算单位数量、止损、加仓价 | 突破触发或信号失效 |
| 触发 | 收盘突破20/55通道 | 建第1单位，立刻写入2N止损 | 触发失败或进入持仓 |
| 持仓 | 已有头寸且未触发退出 | 只按0.5N加盈利仓，更新统一止损 | 触发10/20反向突破或2N止损 |
| 冷却 | 刚止损或连续亏损 | 禁止立刻扩大仓位，只做复盘 | 下一次有效突破且评分达标 |

## 5. N 值与仓位计算

N 是 20 单位真实波幅 TR 的指数移动平均：

```text
TR = max(最高价 - 最低价, abs(最高价 - 前收盘), abs(前收盘 - 最低价))
N = (前一日N * 19 + 当日TR) / 20
```

BTC 日线 N/价格中位数为 3.59%，2N 止损中位距离约 7.17%。因此仓位必须随波动收缩：

```text
账户可亏损金额 = 账户权益 * 单位风险比例
每 BTC 风险 = 2N
第1单位 BTC 数量 = 账户可亏损金额 / 每 BTC 风险
```

建议单位风险：

| 账户/环境 | 单位风险 | 说明 |
| --- | --- | --- |
| 新策略实盘验证 | 0.25%-0.50% | 先验证执行纪律 |
| 普通趋势环境 | 0.50%-1.00% | BTC 单品种建议默认区间 |
| 多周期同向强趋势 | 1.00% | 需要明确止损和无杠杆或低杠杆 |
| 高杠杆/高波动/资金费率异常 | 0%-0.25% | 优先不交易 |

原版“1N=账户1%、2N=2%风险”在 BTC 上应视为上限，不是默认值。

## 6. 入场规则

### 系统1：20日突破

- 做多：收盘价突破过去 20 日最高价。
- 做空：收盘价跌破过去 20 日最低价。
- 退出：出现 10 日反向突破。
- 用途：反应更快，信号更多，更容易遇到假突破。

执行过滤：

- 若上一次系统1突破最终盈利，则下一次系统1信号降级或跳过。
- 若跳过系统1后出现 55 日突破，必须按系统2处理，避免错过大趋势。

### 系统2：55日突破

- 做多：收盘价突破过去 55 日最高价。
- 做空：收盘价跌破过去 55 日最低价。
- 退出：出现 20 日反向突破。
- 用途：主趋势系统，信号少但更适合 BTC 波段趋势。

系统2不使用“上一笔盈利则跳过”的过滤，所有有效突破都要记录。

入场优先级：

1. 周线与日线同向的 55 日突破，优先级最高。
2. 日线趋势过滤支持时，4小时 55 根突破可作为标准执行信号。
3. 日线趋势过滤支持时，4小时 20 根突破只作为提前试探，不得直接满单位。
4. 日线趋势过滤不支持时，所有 4小时突破都降级为观察信号。
5. 信号触发但 N 值处于极端高位时，宁可缩小单位，也不压缩止损。

## 7. 加仓与止损

初始入场后：

1. 建立第 1 单位，止损放在入场价反向 2N。
2. 价格朝盈利方向推进 0.5N，加第 2 单位。
3. 每次新增单位后，把统一保护止损推进到最新加仓价反向 2N。
4. 最多 4 单位；BTC 单品种建议常态最多 2-3 单位，只有周线和日线同向时才允许 4 单位。

多头示例：

```text
第1单位：突破价 P
第2单位：P + 0.5N
第3单位：P + 1.0N
第4单位：P + 1.5N
统一止损：最新加仓价 - 2N
```

禁止事项：

- 亏损加仓。
- 因“快回来了”取消 2N 止损。
- 因错过第 1 单位而在第 3 单位位置追满仓。
- 在 N 值快速扩大时维持原币数不变。

## 8. 离场规则

趋势跟踪的利润来自少数大行情，所以不能用小利润目标过早卖出。

| 系统 | 趋势退出 | 风险退出 | 执行要求 |
| --- | --- | --- | --- |
| 系统1 | 10日反向突破 | 2N止损 | 快进快出，亏损要小 |
| 系统2 | 20日反向突破 | 2N止损 | 忍受利润回吐，等待趋势结束 |

若趋势退出和止损同时接近，优先执行更靠近市场价格、风险更小的规则。出场后不得立刻凭情绪反手，必须等待新突破。

离场后的处理：

- 2N止损退出：记录为风险失败，下一笔同向信号至少降半单位，直到重新突破55单位通道。
- 10/20反向突破退出：记录为趋势结束，可以等待下一次同方向突破，不惩罚系统。
- 提前主观退出：记录为纪律偏差；若连续2次提前退出，暂停新开仓，只允许模拟执行。
- 盈利退出后不预测回调深度，只等待新的系统信号。

## 9. 信号评分

| 项目 | 分值 | 条件 |
| --- | --- | --- |
| 周期一致 | 2 | 周线、日线同向 |
| 趋势过滤 | 2 | 50/300均线支持交易方向 |
| 突破质量 | 2 | 突破前至少横盘/压缩 10 根以上，非连续暴涨后追入 |
| N值稳定 | 1 | N/价格未处于近半年极端高位 |
| 加仓空间 | 1 | 到下一个阻力/支撑至少有 1.5N |
| 资金费率/杠杆 | 1 | 资金费率和强平价不侵蚀 2N 止损 |
| 执行状态 | 1 | 最近 3 笔没有违反止损或跳信号 |

执行阈值：

- 8-10 分：允许标准单位。
- 6-7 分：半单位。
- 低于 6 分：不交易。
- 任何时候，只要止损距离导致账户亏损超过计划风险，分数作废。

评分只决定“是否交易”和“用多大单位”，不允许用来移动止损、取消退出或追加入场理由。分数低于6分但最终盈利，也视为坏交易；分数达标但止损亏损，仍视为正确交易。

## 10. 复盘指标

每 20 笔交易或每月复盘一次，只看这些指标：

| 指标 | 合格方向 | 含义 |
| --- | --- | --- |
| 平均R | 大于0 | 每承担 1R 风险是否有正回报 |
| 利润因子 | 大于1 | 总盈利是否覆盖总亏损 |
| 最大回撤 | 低于预设上限 | 是否还活在系统内 |
| 胜率 | 可低于50% | 海龟法不追求高胜率 |
| 最大连续亏损 | 可承受 | 决定单位风险是否过大 |
| 跳过信号次数 | 越低越好 | 衡量纪律，而不是预测能力 |

如果连续 3 笔亏损，不是立刻改参数，而是先检查：

1. 是否在趋势过滤不支持时交易。
2. 是否把 20 日突破当成系统2重仓。
3. 是否在 N 值扩大后仍用旧仓位。
4. 是否提前止盈导致盈利R倍数不足。
5. 是否漏掉了真正的大趋势信号。

参数迭代规则：

- 少于30笔闭合交易，不优化参数，只复盘执行偏差。
- 30-60笔交易后，只允许调整单位风险、最大单位数和趋势过滤，不改20/55与10/20核心结构。
- 只有当同一问题连续出现两个复盘周期，才允许提出规则修改。
- 任何新增规则必须能用一句话解释，并能在历史样本之外继续执行。
- 禁止为了修复某一笔大亏损而增加特殊例外。

## 11. BTC 执行清单

单笔交易计划卡：

| 字段 | 必填内容 |
| --- | --- |
| 交易模式 | 保守/标准/激进 |
| 触发系统 | 系统1或系统2，日线或4小时 |
| 趋势过滤 | 周线、日线、50/300是否同向 |
| 当前N值 | N、2N距离、N/价格百分比 |
| 单位风险 | 本次每单位账户风险百分比 |
| 入场价 | 突破确认价或执行价 |
| 第1单位数量 | 按2N止损反推BTC数量 |
| 加仓价 | 第2/3/4单位价格 |
| 统一止损 | 当前必须执行的止损价 |
| 趋势退出线 | 10/20反向突破价 |
| 作废条件 | 哪些情况出现就不再交易 |
| 复盘结论 | 正确执行/纪律偏差/规则需观察 |

- [ ] 今天交易的是系统1还是系统2？
- [ ] 当前 N 值是多少，2N 对应账户亏损是多少？
- [ ] 过去 20/55 日高低点在哪里？
- [ ] 10/20 日退出线在哪里？
- [ ] 50/300 趋势过滤是否支持方向？
- [ ] 第 1 单位数量是多少？
- [ ] 第 2/3/4 单位加仓价是多少？
- [ ] 统一止损价在哪里？
- [ ] 若跳空越过止损，账户最大损失估计是多少？
- [ ] 本次交易的信号评分是否达到 6 分？
- [ ] 是否存在资金费率、杠杆强平或流动性风险？
- [ ] 如果这笔亏损，是否仍会执行下一笔信号？

| N值 | 20单位TR的EMA | 波动率标准化 |
| 系统1入场 | 20单位突破 | 快速趋势 |
| 系统1退出 | 10单位反向突破 | 快速止盈/止损 |
| 系统2入场 | 55单位突破 | 主趋势 |

## 13. 禁用与降级规则

- 不把原书多期货分散化结论直接套到单一 BTC。
- 不把 2% 单笔风险当默认仓位；BTC 单品种先用 0.25%-1%。
- 不在突破前预测方向，信号没出现就不提前下注。
- 不因上一笔亏损跳过下一笔有效系统2信号。
- 不为提高回测收益而加入只在少数历史交易中生效的特殊规则。
- 不用固定币数替代 N 值仓位。
- 不用小止盈破坏 10/20 日反向突破退出。

## 14. 一句话版本

用 20/55 日突破发现 BTC 趋势，用 N 值决定买多少，用 2N 止损保证活下来，用 0.5N 只加盈利仓，用 10/20 日反向突破退出；所有规则先服从风险，再服从纪律，最后才谈收益。

# BTC 威科夫操盘法：验证后可复用交易方法论

> 数据基础：Binance `BTCUSDT` 最近两年 K 线，验证区间 2024-06-28 至 2026-06-27。
> 适用对象：BTCUSDT 日线波段交易，4小时用于执行优化，1小时只观察细节；不构成投资建议。

## 1. 方法论定位

这份方法论只保留《威科夫操盘法》中能被 BTC K 线复验、且能写成交易流程的部分：背景、供求代理、努力与结果、SC/ST、Spring、JOC/SOS、UT/SOW、低量回测、危机管理和多时间框架。

“CM/主力”只作为解释语言，不作为下单证据。执行时统一改写为可观察事实：放量推进、放量无进展、缩量测试、跌破收回、突破失败、低量回测、承接、卖压。

## 2. 总流程

威科夫交易不是找单一形态，而是按顺序回答五个问题：

1. 背景：周线是多头、空头还是震荡？
2. 位置：日线处在趋势、区间、下跌末端、上涨末端，还是突破边缘？
3. 行为：当前出现的是 SC/ST、Spring、JOC/SOS、UT/SOW，还是普通噪声？
4. 确认：是否有低量测试、放量跟随、跌回区间、跌破结构？
5. 计划：入场、止损、仓位、退出和 1-5 根危机管理是否写清楚？

硬优先级固定为：风险与退出 > 周线背景 > 日线位置 > 价量确认 > 4小时执行。低优先级信号不能推翻高优先级风险。

## 3. 数据定义

| 概念 | BTC 执行定义 | 用法 |
| --- | --- | --- |
| 高量 | 当前量 >= 20根均量的1.25-1.50倍 | 确认突破、识别异常、观察卖压/承接 |
| 低量 | 当前量 <= 20根均量的0.80-0.95倍 | 二次测试、回测、缩量回调 |
| 宽幅 | 当前振幅 >= 20根均振幅的1.05-1.25倍 | 识别推进、SC/BC、SOW |
| 小实体 | 实体 <= 全振幅25% | 努力无结果、分歧、吸收或派发 |
| 强实体 | 实体 >= 全振幅40%-55% | 推进、突破或跌破确认 |
| 收盘位置 | 收盘接近高位或低位 | 判断当根由需求还是供应收尾 |

日线样本中，“努力有结果/放量宽幅实体”的顺向命中率为 51.85%；“努力无结果/放量窄幅或小实体”的当前方向失效率为 57.14%。结论是：努力与结果可以做警报，但不能脱离位置。

### 3.1 证据等级

两年 BTC 样本不能把所有威科夫术语都验证成“稳定优势”。执行时按证据等级使用：

| 等级 | 条件 | 用法 |
| --- | --- | --- |
| A级 | 多周期一致、样本不少、结果与原理同向 | 可进入评分和仓位模型 |
| B级 | 日线样本有限，但4小时/1小时和原理同向 | 只做候选，必须等确认 |
| C级 | 样本很少或结果不稳定 | 只做提醒，不作为入场理由 |
| 不可验证 | 涉及主力身份、新闻、订单流来源 | 只能作为叙事，不能写进交易触发 |

本方法的 A/B 级核心只有三类：周线背景过滤、SC 后二次测试/确认流程、Spring 或 JOC 后的危机管理。单根 K 线形态默认都是 C 级。

## 4. 背景判断

周线只回答能不能做日线波段重仓：

| 周线状态 | 判断 | 日线动作 |
| --- | --- | --- |
| 多头 | 周收盘在20周均线上方，且未出现连续 UT/SOW | 允许标准多头计划 |
| 中性 | 周线横盘，日线处在大区间内 | 只做区间边缘和确认信号 |
| 空头 | 周收盘在20周均线下方，或放量跌破结构 | 禁止因单根 Spring/长下影重仓抄底 |

日线 JOC 与周线多头背景同向时，本次样本命中率为 64.29%，平均结果 +0.36%，样本 14 个。周线背景不是装饰，是过滤器。

### 4.1 周线-日线动作矩阵

| 周线背景 | 日线信号 | 动作 |
| --- | --- | --- |
| 多头 | JOC + 回测不跌回 | 标准计划，按评分决定仓位 |
| 多头 | Spring 确认 | 试探或半仓，跟随确认后加仓 |
| 多头 | UT/SOW | 先保护利润，不新增多头 |
| 中性 | JOC 或 Spring | 只做小仓，目标降低到区间上沿/下沿 |
| 中性 | SC/BC 单根信号 | 只观察，等待右手边 |
| 空头 | Spring、长下影、SC | 禁止重仓抄底，只允许记录候选 |
| 空头 | JOC | 降级为反弹信号，必须等周线重新转强 |

## 5. 吸筹流程的可交易版本

吸筹不要写成“主力在买”，而要写成一组可见行为：

1. 先有下跌末端或区间下沿。
2. 出现 SC 或放量止跌，只标记“供应释放/承接候选”。
3. 等 ST：回到前低附近时成交量显著小于 SC，振幅变小，不能有效跌破。
4. 等 Spring 或 SOS/JOC：跌破收回，或放量离开区间。
5. 优先在低量回测、4小时转强、或日线重新收上关键位后执行。

本次验证中，SC 后缩量二次测试的日线命中率为 62.50%；SC 后放量复测不应视为成功测试。结论：不要在第一根恐慌 K 线抄底。

## 6. Spring 交易规则

Spring 是“跌破支撑后收回”的测试，不是“跌破就买”。

| 类型 | BTC 定义 | 动作 |
| --- | --- | --- |
| 低量 Spring | 小幅跌破20根低点后收回，成交量低于或接近均量 | 候选，可等4小时确认或下一根不破低点 |
| 放量 Spring | 跌破收回但成交量高 | 只观察，必须等二次测试或连续上行确认 |
| 确认 Spring | Spring 后1-3根突破信号 K 高点且不破低点 | 可进入交易计划 |
| 失败 Spring | Spring 后1-3根跌破信号 K 低点 | 退出、禁止补仓 |

日线低量 Spring 样本只有 1 个，命中率 100.00% 只能作参考；日线 Spring 后 1-3 根向上确认命中率为 70.00%。执行时以确认优先于猜底。

## 7. JOC/SOS 交易规则

JOC/SOS 是本方法最核心的多头触发：

1. 日线有明确20-55根震荡区或前高压力。
2. 收盘突破区间上沿或20根高点。
3. 成交量 >= 20根均量1.25倍。
4. 实体不小，收盘靠近高位。
5. 周线不能明显逆向。

日线 JOC/SOS 放量突破上沿命中率为 54.55%；JOC 后低量回测不跌回区间命中率为 50.00%。标准仓位优先等低量回测；突破当根只允许在止损很近、周线同向时试探。

## 8. 派发与退出规则

派发不要写成“主力在卖”，而要写成“卖压代理行为”：

| 信号 | 定义 | 动作 |
| --- | --- | --- |
| UT | 上冲20根高点但收盘跌回 | 多头减仓或禁止追高 |
| 放量/长上影 UT | UT 同时高量或长上影 | 优先退出试探仓 |
| SOW | 放量强实体跌破20根低点 | 退出，等待 LPSY/反弹弱再评估 |
| JOC 失败 | 突破后1-8根放量跌回区间 | 危机管理退出 |
| 跌破50日线且放量 | 趋势保护失败 | 退出波段仓 |

日线 UT 后 1-3 根放量跌破的反向命中率为 80.00%。UT/SOW 可用于退出，但不机械做空；做空还需要资金费率、波动和止损距离验证。

## 9. 入场模型

只允许三类入场：

### A. JOC 低量回测入场

条件：

1. 周线多头或中性，不是明确空头。
2. 日线放量 JOC/SOS 突破区间。
3. 回测原阻力时成交量 <= 20根均量0.90倍，收盘不跌回区间。
4. 4小时重新转强，或日线下一根收回回测 K 高点。
5. 止损放在回测低点或 JOC K 线低点下方。

### B. Spring 确认入场

条件：

1. 日线处于上升趋势回调、区间下沿，或疑似吸筹右手边。
2. 小幅跌破前低后收回，不能是周线空头中的单根诱多。
3. 若放量，必须等二次测试缩量；若低量，可等 4小时更高低点。
4. 入场后 1-5 根必须出现向上跟随，否则退出。
5. 止损放在 Spring 低点外侧。

### C. 缩量回调后延续入场

条件：

1. 周线多头，日线在50/200日均线上方。
2. 回调阶段量能下降，没有放量跌破结构。
3. 4小时放量收回短压，或日线重新收阳突破回调高点。
4. 止损放在回调低点下方。

其他信号一律降级为观察，包括单根 SC、单根长下影、单根高量小实体、缩量突破和周线空头中的 Spring。

### 9.1 入场优先级

同一天出现多个信号时，按优先级处理：

1. 先处理退出信号：UT、SOW、JOC 失败、放量跌破结构。
2. 再处理周线背景：周线空头时，多头信号全部降级。
3. 再处理入场触发：JOC 回测优先于突破当根，确认 Spring 优先于放量 Spring。
4. 最后处理4小时执行：4小时只能优化入场，不能推翻日线风险。

若退出信号和入场信号同一天出现，默认不交易，等下一根日线确认。

## 10. 危机管理

进场后 1-5 根必须甄别。如果市场行为否定入场依据，先退出，不争辩。

| 入场依据 | 正常跟随 | 危机信号 |
| --- | --- | --- |
| JOC | 突破后不跌回区间，回测缩量 | 放量跌回区间、长上影、连续无需求 |
| Spring | 不破低点，逐步抬高低点 | 1-3根跌破 Spring 低点 |
| 缩量回调 | 重新放量上攻 | 放量跌破回调低点 |

JOC 后放量跌回区间不应被解释为加仓机会；即使后续可能重新走强，也必须先按危机管理处理，等市场重新给出确认。危机管理比预测更重要。

## 11. 仓位规则

| 场景 | 仓位 |
| --- | --- |
| 周线多头 + 日线 JOC + 低量回测成功 | 50%-100%计划仓位 |
| Spring 确认且止损很近 | 20%-50%计划仓位 |
| 放量 Spring 后二次测试成功 | 10%-30%试探仓，确认后再加 |
| 周线中性 | 标准仓位减半 |
| 周线空头、缩量突破、单根 SC/长下影 | 默认 0%，只观察 |

单笔计划亏损控制在账户权益的0.5%-1.5%。先定止损，再用止损距离反推数量；止损太远就放弃，不把止损内移到“看起来舒服”的位置。

头寸计算固定为：

| 步骤 | 公式 |
| --- | --- |
| 单笔最大亏损 | 账户权益 × 0.5%-1.5% |
| 每 BTC 风险 | 入场价 - 止损价，做空反向计算 |
| 可交易数量 | 单笔最大亏损 / 每 BTC 风险 |
| 第一目标 | 至少 1.5R，优先取前高/区间等幅/上方供应区中最近者 |
| 加仓条件 | 原仓已有浮盈，且新止损不扩大总计划亏损 |

若止损距离超过日线 ATR 的 1.5-2.0 倍，默认等待回测或放弃，不用扩大仓位补偿“错过感”。

## 12. 信号评分

入场前打分，低于6分不交易：

| 项目 | 分值 | 条件 |
| --- | --- | --- |
| 周线背景 | 2 | 周线同向或至少不逆向 |
| 日线位置 | 2 | 区间右手边、突破边缘、趋势健康回调 |
| 价量行为 | 2 | JOC/SOS、确认 Spring、低量回测之一 |
| 确认质量 | 2 | 放量跟随、回测缩量、不跌回区间 |
| 风险收益 | 1 | 第一目标至少1.5R |
| 执行周期 | 1 | 4小时同向，未出现失败信号 |

8-10分允许标准仓位；6-7分只允许半仓或试探；低于6分不交易。出现 UT、SOW、JOC 失败或跌破结构时，评分自动失效。

## 13. 执行模板

| 字段 | 内容 |
| --- | --- |
| 周线背景 | 多头/中性/空头 |
| 日线阶段 | 吸筹右手边、趋势回调、JOC突破、派发警报等 |
| 理论依据 | 供应枯竭、需求确认、低量回测、突破失败等 |
| 价量证据 | 成交量倍数、实体比例、收盘位置、是否跌回区间 |
| 入场触发 | JOC回测、Spring确认、4小时转强 |
| 止损位置 | Spring低点、回测低点、JOC低点或结构低点 |
| 计划亏损 | 账户权益百分比 |
| 第一目标 | 前高、区间等幅、2R或上方阻力 |
| 危机管理 | 1-5根内哪些行为触发退出 |
| 复盘结果 | 信号评分、是否等确认、是否按止损执行 |

## 14. 每日检查清单

- [ ] 周线是否支持日线方向？
- [ ] 日线处于趋势、区间、右手边，还是末端风险区？
- [ ] 当前成交量相对20根均量是高、低还是正常？
- [ ] 是否出现努力无结果？
- [ ] 是否出现 SC/ST、Spring、JOC/SOS、UT/SOW？
- [ ] 若是 JOC，是否有低量回测？
- [ ] 若是 Spring，后续 1-3 根是否确认，还是跌破失败？
- [ ] 是否存在放量跌回区间或放量跌破结构？
- [ ] 止损在哪里？若错了账户亏损是多少？
- [ ] 信号评分是否达到6分以上？

## 15. 禁用与降级

- 不用“主力正在吸筹/派发”作为下单理由，只记录可见行为。
- 不在单根 SC 或长下影出现时直接抄底。
- 不把放量 Spring 当成低风险买点，必须二次确认。
- 不追周线空头中的日线缩量突破。
- 不在 JOC 后放量跌回区间时加仓。
- 不把 UT/SOW 机械等同于做空信号，除非另有做空系统和风险约束。
- 不用新闻解释亏损交易，除非事前已写入计划。

出现以下任一情况，暂停新开仓：

- 连续3笔亏损，且至少2笔低于6分。
- 最近5笔交易中有2笔违反“先退出再等确认”。
- 月度已实现亏损达到预设上限。
- 简化规则连续失效，实际交易仍想提高仓位。

## 16. 复盘与迭代

每20笔交易或每月复盘一次，只看五个问题：

1. 亏损是否来自低于6分的信号？
2. 是否在 SC、长下影、放量 Spring 上提前抄底？
3. 是否等到了 JOC 后低量回测？
4. 危机管理是否在 1-5 根内执行？
5. 盈利单是否遵守了 UT/SOW 或放量跌破结构的退出？

本次简化“日线威科夫组合规则”总收益 -10.98%、最大回撤 -12.37%、在场时间 7.95%；同期日线买入持有总收益 0.36%、最大回撤 -52.03%。这说明威科夫方法的可复用价值主要在过滤、等待确认和危机管理。

因此，本方法不是已经证明盈利的自动化系统，而是一套人工决策框架。正式实盘前至少完成20笔纸面交易或小仓交易，且必须满足：

- 低于6分的交易占比低于10%。
- 每笔亏损都在计划亏损内。
- 没有单根 SC、长下影或放量 Spring 直接重仓抄底。
- JOC 失败时能先退出，而不是把失败解释成洗盘。
- 20笔样本后仍能保持正期望或可接受回撤。

## 17. 一句话版本

周线先定背景，日线只交易 JOC 低量回测、确认 Spring 和趋势缩量回调；SC/BC 只开启观察，UT/SOW 优先退出，所有入场都必须在 1-5 根内接受市场复核。
